Manuseio de arquivos

Em algum momento da criação de seu jogo será necessário armazenar informações externamente e ser capaz de recuperar essas informações. Por exemplo, você pode desejar armazenar detalhes de login, ou pontuações, ou talvez a progressão do jogador através de seu níveis. Seja qual for o motivo, você precisará saber como utilizar as diferentes funções de arquivo que estão disponíveis para você ao trabalhar com o GameMaker Studio 2.

É muito importante perceber que o GameMaker Studio 2 é Sandboxed por padrão, o que significa que há uma série de limites e pequenas complicações ao usar as funções do sistema de arquivos que estão disponíveis, e vale a pena notar também que o arquivo funcionará de forma ligeiramente diferente dependendo da plataforma que você está visando. Para mais informações, consulte a seção seguinte:

  • O sistema de arquivo

Você pode desligar o sandboxing nos alvos do desktop(Windows, MacOS e Ubuntu(Linux)) a partir das Opções de Jogo para a plataforma escolhida, significando que você pode então salvar e carregar de qualquer lugar, dentro dos limites das permissões do sistema operacional.

Se você recebeu permissão da Steam para usar sua API dedicada com seu jogo, você verá que há também um novo conjunto de funções de arquivo disponíveis para você. Estas são explicadas na seção Steam API deste manual.

Finalmente, você também deve observar que existem algumas diferenças distintas entre certas plataformas alvo que seriam levadas em consideração ao utilizar as funções de arquivo. Estas são:

  • iOS: No iOS, se você estiver carregando um arquivo incluído de uma pasta, ou seja: com um caminho algo como "Backgrounds/background1.png", você NÃO precisa incluir a pasta como parte do caminho e simplesmente fornecer o nome do arquivo é suficiente. Em todas as outras plataformas, você deve fornecer a(s) pasta(s) como parte do caminho para o arquivo.
  • macOS e Ubuntu (Linux): No MacOS e Ubuntu, os nomes de arquivos e caminhos são sensíveis a maiúsculas e minúsculas nestas plataformas, e assim, para evitar qualquer problema potencial, o GameMaker Studio 2 fará com que automaticamente todos os nomes/caminhos usem minúsculas por padrão.
  • HTML5: Em HTML5, salvar e carregar de e para o armazenamento local funcionará como explicado dentro das diferentes páginas de funções, entretanto, se você estiver carregando um arquivo de um servidor, você só deve usar a função buffer_load_async(), pois o carregamento síncrono foi depreciado na maioria dos navegadores e eventualmente será obsoleto. Observe também que você não pode criar ou destruir diretórios no armazenamento local do navegador.

  • Codificação e Hashing

Observe também que há algumas funções básicas de arquivo para salvar jogos disponíveis nas páginas seguintes (esteja ciente de que estas funções foram projetadas para que os iniciantes tenham um sistema básico de salvamento pronto para funcionar o mais rápido possível, mas para projetos mais complexos, você deve criar seu próprio sistema de salvamento usando as funções listadas nas seções acima):

  • jogo_load
  • jogo_load_buffer
  • jogo_save
  • jogo_save_buffer

Finalmente, você pode encontrar funções de buffer mais avançadas que podem ser usadas para carregar e salvar dados nas páginas seguintes:
