Arquivos de texto

Arquivos de texto são uma maneira útil de armazenar grandes quantidades de dados externamente de seu jogo. Ao contrário dos arquivos ini, você pode ter até 32 arquivos de texto abertos ao mesmo tempo, e você pode ler e escrever grandes pedaços de dados para eles também. No entanto, cabe a você decidir como esses dados devem ser estruturados, já que não há uma estrutura definida de "seção" ou "chave" como nos arquivos *.ini.

Observe que para jogos com localizações é muito importante que as letras acentuadas possam ser lidas a partir de arquivos externos. Isto significa que você deve criar primeiro o arquivo txt no formato UTF8 e depois adicioná-lo ao GameMaker Studio 2 como um arquivo incluído para que seja exportado durante a execução do jogo e utilizado no lugar do arquivo txt padrão no formato ANSI que é criado pelo arquivo GameMaker Studio 2 funciona quando nenhum arquivo é fornecido anteriormente. Desta forma, você pode ler e escrever corretamente com todos os sotaques e cartas não romanas mantidas.

Existem as seguintes funções que tratam de arquivos:

  • file_text_open_read
  • file_text_open_write
  • file_text_open_append
  • file_text_open_from_string
  • file_text_read_real
  • file_text_read_string
  • file_text_readln
  • file_text_write_real
  • file_text_write_string
  • file_text_writeln
  • file_text_eoln
  • file_text_eof
  • file_text_close
