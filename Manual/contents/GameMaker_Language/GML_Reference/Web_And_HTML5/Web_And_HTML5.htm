Web e HTML5

OGameMaker Studio 2 oferece várias funções para interagir com a Internet ou com um servidor, e elas podem ser usadas em todas as plataformas alvo disponíveis. Estas funções são:

  • url_open
  • url_open_ext
  • url_open_full
  • url_get_domain
  • http_request
  • http_get
  • http_get_file
  • http_post_string

Além das funções gerais listadas acima, o alvo HTML5 tem algumas funções adicionais da web que são especificamente projetadas para serem usadas pelos jogos que o visam:

  • altura_do_browser
  • largura_do_browser
  • browser_input_capture
  • webgl_enabled
  • clickable_exists
  • clickable_add
  • clickable_add_ext
  • clickable_change
  • clickable_change_ext
  • clickable_set_style
  • clickable_delete
  • evento_analítico
  • análise_evento_extra_analítico
  • os_browser
  • http_get_request_request_crossorigin
  • http_set_request_crossorigin

Além destas funções para o módulo de destino HTML5, há também uma variável especial JavaScript que pode ser usada naquelas raras ocasiões em que o conteúdo da pasta html5game é armazenado em um lugar completamente diferente (URL). Normalmente este caminho é codificado no arquivo JavaScript e é relativo ao local de onde o JavaScript é chamado, mas você pode definir a variável g_GameMakerHTML5Dir antes da chamada para o roteiro do jogo e o jogo será executado a partir do local recém-definido. Por exemplo:

<script>var g_GameMakerHTML5Dir = "http://mygamealternatedirectory.com/";</script>

Também vale a pena notar que você pode sinalizar funções de script usando o prefixo gmcallback_ o que significa que elas não serão ofuscadas quando você compilar o projeto para HTML5. Isto é muito uisqueiro ao usar as funções acima listadas clickable_* (para mais informações, por favor, veja aqui).
