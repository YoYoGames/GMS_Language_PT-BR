clickable_add

Com esta função você pode criar um ícone personalizado, clicável, que atua como um "botão" fora da tela do jogo, mas dentro da própria janela do navegador. A função retornará um valor de id que deve ser armazenado em uma variável para ser usada em todos outras funções do botão que fazem referência ao novo ícone. Esta função é muito útil para evitar que o navegador crie uma janela pop-up ao clicar no botão, pois tudo isso é feito através do próprio GameMaker Studio 2.

Primeiro você deve definir a posição, que é relativa ao (0,0) da janela do navegador (canto superior esquerdo) e depois dar a entrada da página de textura para o sprite que você deseja que o ícone do botão tenha (veja: sprite_get_tpe()). Você pode então especificar um URL para abrir ou um script (do Navegador de Recursos do GameMaker Studio 2) para ser aberto ou executado. O parâmetro "alvo" que você especifica é o mesmo que o valor "nome" padrão do JavaScript quando você usar o método open( ) (estar ciente de que tudo menos "_self" pode resultar no bloqueio do navegador, ou perguntar ao usuário se ele deseja permitir) e o argumento "params" é o mesmo que o parâmetro "specs" do JavaScript para controlando quais propriedades a nova janela/tabela deve exibir (nem todos os navegadores podem suportar todas as características). Se você estiver usando o ícone para executar um script interno, estes dois últimos argumentos podem ser deixados como cordas vazias "".

Os alvos válidos são:

Meta Descrição
em branco Abre o documento vinculado em uma nova janela ou aba.
_self Abre o documento vinculado no mesmo quadro em que foi clicado (este é o padrão).
pai Abre o documento vinculado no quadro superior.
_topo Abre o documento vinculado no corpo completo da janela.

Os parâmetros válidos são:

Parâmetros Descrição
height=[px]''. A altura da janela, com o valor mínimo sendo 100.
width=[px]''. A largura da janela, com o valor mínimo sendo 100.
esquerda=[px]''. A posição esquerda da janela.
top=[px]''. A posição superior da janela (apenas IE).
localização=[booleano]''. Se deve ou não exibir o campo de endereço (o padrão é 1).
menubar=[boolean]''. Se a barra de menu deve ou não ser exibida (o padrão é 1).
redimensionável=[booleano]''. Se a janela é redimensionável ou não (o padrão é 1).
scrollbars=[boolean]''. Se deve ou não exibir barras de rolagem (o padrão é 1).
status=[boolean]''. Adicionar ou não uma barra de status (o padrão é 1).
titlebar=[boolean]''. Se a barra de título deve ou não ser exibida. Isto é ignorado a menos que a aplicação de chamada seja uma aplicação HTML ou uma caixa de diálogo confiável (o padrão é 1);
barra de ferramentas=[booleano]''. Se a barra de ferramentas do navegador deve ou não ser exibida (o padrão é sim).


Também vale a pena notar que se o argumento URL que você passa é o nome de uma função do script (ou seja: não começa com "http://") E o script é chamado gmcallback_*. então, você pode obter feeds diretos em um evento html5. Os nomes das funções do GameMaker Studio 2 não são ofuscados se começarem com gmcallback_.

Sintaxe:

clickable_add(x, y, tpe, url, target, params);

Argumento Descrição
x A posição x dentro da janela.
y A posição y dentro da janela.
tpe A entrada da página de textura para o duende a ser utilizado.
url O URL (endereço do site) para o link, ou o roteiro a ser executado.
alvo Esta é a área de destino para abrir a URL em (ver descrição).
param Os diversos parâmetros para o ícone (ver descrição).

Devoluções:

Real

Exemplo:

home_but = clickable_add(32, 32, sprite_get_tpe(spr_MS_Home, 0), "http://macsweeney_games.com", "_blank", "width=700, height=500, menubar=0, toolbar=0, scrollbars=0");

O código acima cria um ícone DOM clicável na posição (32, 32) da página em que a tela do jogo está rodando. O ícone usa o sprite referenciado da página de textura como "spr_MS_Home" e quando clicado, o ícone abrirá uma nova janela para a URL especificada e com as propriedades definidas para aquela janela.
