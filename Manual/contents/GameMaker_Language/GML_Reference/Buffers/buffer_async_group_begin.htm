buffer_async_group_begin

Esta função é chamada quando se deseja começar a salvar a partir de múltiplos buffers para múltiplos arquivos. O "groupname" é uma string e será usado como nome do diretório onde os arquivos serão salvos, e deve ser usado como parte do arquivo ao carregar os arquivos de volta ao IDE mais tarde (usando qualquer um dos caminhos do buffer_load() funções). Esta função é apenas para uso com o buffer_save_async() função e você também deve terminar a definição de salvar chamando buffer_async_group_end() Caso contrário, os arquivos não serão salvos.

Note que para as plataformas de console (como PS4, por exemplo), o "groupname" será usado como a descrição do slot salvo, e usar esta função pode ajudá-lo a evitar que a IU mostre para cada arquivo que está sendo salvo. Observe também que quando Usando o UWP, você pode percorrer seus jogos salvos entre as vitrines nativas da XBox e da Microsoft, criando um grupo que é o mesmo que um grupo ERA Xbox, por exemplo:

if os_type == os_uwp
{
buffer_async_group_begin("root/yourgroupname");
}
mais
{
buffer_async_group_begin("yourgroupname");
}

Sintaxe:

buffer_async_group_begin(groupname);

Argumento Descrição
nome de grupo O nome do grupo (como um fio).

Devoluções:

N/A

Exemplo:

buffer_async_group_begin("SaveGame");
save1 = buffer_save_async(buff1, "Player_Save1.sav", 0, 16384);
save2 = buffer_save_async(buff2, "Player_Save2.sav", 0, 16384);
save3 = buffer_save_async(buff3, "Player_Save3.sav", 0, 16384);
save4 = buffer_save_async(buff4, "Player_Save4.sav", 0, 16384);
buffer_async_group_end();

O código acima inicia um grupo tampão e depois o define para salvar 4 arquivos de forma assíncrona. A definição do grupo é então terminada (momento em que a gravação começará).
