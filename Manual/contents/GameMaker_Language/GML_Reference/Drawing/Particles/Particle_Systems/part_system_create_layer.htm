part_system_create_layer

Esta função criará um novo sistema de partículas em uma determinada camada. Você dá o ID da camada única como retornado pela função layer_create() ou o nome de a camada a ser usada como um fio - por exemplo, "exemplo_camada" - e depois sinalizar o sistema como sendo persistente ou não. Se o sistema não for sinalizado como persistente, será automaticamente destruído no final da sala em que foi criado. (isto é o mesmo que se você tivesse chamado a função part_system_destroy() e também destruirá quaisquer emissores associados com o sistema). Entretanto, quando marcado como persistente, o sistema será transportado para a próxima sala quando a sala for trocada, e se a sala seguinte não tiver uma camada com o mesmo nome ou profundidade que a atribuída, então uma nova camada será criada para o sistema que está persistindo nas salas, e será nomeada igual à camada original. Ao trocar de sala, se houver outra camada nas seguintes salas com o mesmo nome, então a instância persistente será atribuída à camada com o mesmo nome, independentemente da profundidade da camada. Finalmente, se um sistema persistido se mover para uma sala com uma camada à mesma profundidade que a camada em que o sistema foi criado, ele não será atribuído a esta camada, mas em vez disso uma nova camada será criada à mesma profundidade (com o mesmo nome que o original camada).

A função retornará um valor de identificação único para o sistema de partículas que deve ser usado em todas as outras chamadas de função em que você precisa dar uma identificação do sistema.

IMPORTANTE! Se você sinalizar o sistema de partículas como persistente, então ele (e quaisquer emissores atribuídos a ele) precisará ser limpo manualmente usando as funções de destruição apropriadas quando não estiver em uso, caso contrário você corre o risco de um vazamento de memória que irá negativamente impactar seu jogo final.

Sintaxe:

part_system_create_layer(layer_id, persistente);

Argumento Descrição
camada O valor de identificação da camada ou string para atribuir o sistema de partículas (pode ser qualquer tipo de camada)
persistente Marcar o sistema de partículas como persistente (definido como verdadeiro) ou não (definido como falso)

Devoluções:

Real (valor único de identificação do sistema de partículas)

Exemplo:

global.p_sys = part_system_create_layer("effects_layer", verdadeiro);

O código acima criará um novo sistema de partículas na camada dada e o sinalizará como persistente sobre as salas subseqüentes. A identificação para o sistema de partículas é armazenada em uma variável de escopo global para referência futura.
