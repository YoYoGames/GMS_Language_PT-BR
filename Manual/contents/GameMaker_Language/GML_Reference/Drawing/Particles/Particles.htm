Partículas

Para coisas complexas no GameMaker Studio 2 você normalmente teria um objeto e criaria instâncias desse objeto ao redor da sala. Entretanto, para efeitos gráficos, isto pode ser caro, pois cada instância vem com um "custo" no processamento devido às variáveis que ela contém e aos códigos que ela tem nos diferentes eventos. Você pode reduzir este custo se voltando para camadas de mapas de azulejos e camadas de ativos para desenhar suas necessidades gráficas, mas estas são geralmente estáticas e não podem ser alteradas ou movidas muito. No entanto, existe uma outra opção para desenhar efeitos gráficos rápidos mas versáteis em seus jogos, e isso é usar partículas.

As partículas são recursos gráficos que possuem certas propriedades definidas dentro de um sistema de partículas. Estas propriedades não podem ser manipuladas diretamente para partículas individuais, mas são alteradas através do código que é utilizado para definir a partícula individual e o sistema ao qual ela pertence. Eles são muito úteis para criar efeitos lindos e brilhantes (ou sutis e discretos!) como explosões, decalques, chuva, neve, campos de estrelas e escombros em um jogo sem a CPU que o uso de instâncias tem.

A configuração básica para um sistema de partículas segue três passos, sendo o terceiro passo opcional, dependendo de como você deseja criar seus efeitos de partícula dentro da sala:

  • Criar um sistema de partícula: O sistema de partículas é como um recipiente que usaremos para manter nossos diferentes tipos de partículas prontos para uso. Usamos código para definir uma série de aspectos visuais e comportamentos para nossas partículas, e depois os colocamos no "container" para que possamos retirá-los e utilizá-los quando e onde precisarmos mais tarde.
  • Criar tipos de partículas: Os tipos de partícula são o próprio efeito gráfico. Você pode ter muitos tipos diferentes, cada um com sua própria gama de cores, alfas, tamanhos e movimentos, mas é importante notar que você não tem controle sobre as partículas individuais. Você define uma série de parâmetros e a partícula será criada para ter uma distribuição aleatória de comportamentos escolhidos a partir deles.
  • Criar emissores: Os emissores são uma opção que pode ser usada para explodir ou emitir partículas a partir de limites muito claramente definidos. Eles são opcionais porque você pode criar partículas a partir de qualquer instância usando o part_particles_create() e part_particles_create_colour() mas nem sempre são adequadas para todas as situações.

Embora as partículas sejam uma excelente ferramenta para criar efeitos, elas vêm com certas restrições e boas práticas que precisam ser seguidas, a menos que você queira que seu jogo sofra de mau desempenho ou até mesmo de um possível colapso:

  • Os sistemas de partículas, partículas e emissores ocupam a memória e como tal você deve ser muito cuidadoso na forma de usá-los, pois é muito fácil causar um vazamento de memória que irá diminuir a velocidade e, eventualmente, quebrar seu jogo. Uma maneira de lidar com isto é ter um sistema global com tudo definido no início do jogo e removido no final, mas se você quiser um sistema dinâmico, então cada partícula e emissor (e o próprio sistema) deve ser destruído no momento em que não for necessário.
  • As partículas podem ser rápidas e leves na CPU, mas ainda requerem algum processamento e, portanto, não se deve ter 40.000 delas estourando pela tela de cada vez. Limite-as àquelas que são necessárias para obter um efeito específico e não mais.
  • Se você definir seu próprio duende de partículas em vez de usar um dos 14 sprites incluídos, você deve tentar mantê-los tão pequenos quanto possível para alcançar o efeito desejado.
  • As partículas não interagem com nada. Se você precisar delas para ter qualquer tipo de interação com o usuário ou qualquer outra instância em seu jogo, você deve estar olhando para o uso de instâncias em vez disso, pois as partículas são puramente gráficas.
  • Mesmo que não haja limite técnico para a quantidade de sistemas, emissores e partículas que você pode criar em um jogo, você deve tentar limitar tudo ao mínimo possível para manter o uso da memória o mais baixo possível.
  • Em dispositivos móveis, tome cuidado com as partículas, pois o desenho pode ser lento se cobrir uma grande área da tela (o desenho excessivo em dispositivos móveis é uma das principais causas de desaceleração).
  • Com HTML5 não há mistura aditiva, e a menos que você tenha WebGL habilitado, você também não pode ter mistura de cores.

A maneira mais fácil de criar partículas em seu jogo com o GameMaker Studio 2 é usar o mecanismo de efeitos embutido. Estes efeitos são criados usando um sistema interno de partículas que é basicamente um método muito rápido para desenhar efeitos gráficos. somente e como tal, você não precisa se preocupar com todos os detalhes (como gerenciamento de memória) ao usar estas funções. Você simplesmente especifica o tipo de efeito, a posição onde ele deve ser criado, o tamanho que você quer que ele seja e, finalmente, sua cor, então deixe o GameMaker Studio 2 fazer todo o trabalho.

As seguintes funções existem para criar os efeitos de partículas incorporadas:

  • effect_create_below
  • effect_create_above
  • effect_clear

Embora estes efeitos sejam de escopo e personalização limitados, eles ainda podem ser usados para criar alguns efeitos simples e grandes com muito pouco esforço. Por exemplo, ao criar uma pequena nuvem de fumaça cinza abaixo de um míssil em movimento em cada passo, um efeito bastante convincente O rastro de fumaça é criado, portanto, mesmo quando você é um especialista em partículas, compensa lembrar que estes efeitos existem, pois ainda podem lhe poupar algum tempo.
