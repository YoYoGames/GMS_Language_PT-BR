gamepad_get_option

Esta função pode ser usada para recuperar o valor da opção dada. Você fornece o "slot" do controle de jogo para verificar, assim como a cadeia de opções e a função retornará o valor associado. A cadeia de opções disponíveis dependerá da plataforma que o projeto está sendo executado, como listado abaixo:

Opção Plataforma Descrição Devoluções
"permitir_rotação" tvOS Retorna se o Siri Remote será bloqueado por rotação (ou seja: só pode ser usado verticalmente) ou não. Quando destravado, a rotação do controle mudará a entrada, de forma que uma rotação de 90º para a esquerda fará o lado "direito" do toque retorno da almofada "para cima", etc... de modo que toda a entrada será relativa à orientação do dispositivo Booleano
"dpad_absolute" tvOS Retorna se o touchpad Siri Remote aceita entrada absoluta, ou entrada relativa. No modo absoluto, o centro do touchpad é considerado a posição (0,0) - ou seja: onde o bastão analógico está em repouso ou onde o centro do dpad estaria - e o movimento em torno desta posição gerará entrada. Quando definida como relativa, então a posição (0,0) será considerada como estando onde quer que o toque inicial na almofada seja feito e toda a entrada será gerada a partir dessa posição, não o centro do bloco. Booleano

Sintaxe:

gamepad_get_option(dispositivo, nome_opção_opção);

Argumento Descrição
dispositivo Qual dispositivo de controle de jogo "slot" a ser verificado.
option_name O nome da opção para obter o valor de (um fio, veja a tabela acima)

Devoluções:

Int, Bool, Real, String, N/A (depende da plataforma)

Exemplo:

for(var i = 0; i < 12; ++i;)
{
if gamepad_is_connected(i) && gamepad_get_description(i) == " tvOS SiriRemote"
{
if gamepad_get_option(i, "allow_rotation") == falso
{
gamepad_set_option(i, "allow_rotation", true);
}
}
}

O código acima passa por todos os slots do gamepad e verifica a existência do "Siri Remote" na plataforma tvOS. Se for detectado, ele então define o controle remoto para permitir a rotação de entrada.
