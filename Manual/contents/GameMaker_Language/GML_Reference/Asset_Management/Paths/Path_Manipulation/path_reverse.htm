path_reverse

Com esta função é possível inverter a ordem na qual os pontos individuais do caminho são numerados, assim, por exemplo, se o caminho tiver 5 pontos, o ponto 0 se tornaria o ponto 4, o ponto 1 seria o ponto 3 e o ponto 2 não seria alterado. A posição real de os pontos permanecem os mesmos, apenas a ordem em que são processados é alterada. Esta função muda o ativo do caminho real, e assim afetará permanentemente como o caminho é usado por todas as instâncias no jogo, a partir do momento em que o é utilizada até o final do jogo. Se isto não for o que você precisa, então você deve usar uma função como path_duplicate() para criar uma cópia do caminho primeiro, então chame esta função sobre o bem duplicado (não esqueça de ligar para path_delete() sobre o bem quando ele não for mais necessário).

Sintaxe:

path_reverse(index);

Argumento Descrição
índice O índice do caminho para a mudança.

Devoluções:

N/A

Exemplo:

path_reverse(pth_AI);

Isto inverteria a ordem na qual todos os pontos do caminho dado são processados.
