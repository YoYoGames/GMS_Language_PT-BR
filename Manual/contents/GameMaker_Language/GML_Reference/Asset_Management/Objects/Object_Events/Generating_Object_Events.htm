Eventos de geração de objetos

OGameMaker Studio 2 é completamente guiado por eventos, e que todas as ações acontecem como resultado de certos eventos que são acionados em uma ordem específica ou por uma ocorrência específica dentro do jogo. Há uma série de eventos diferentes, cada um com uma tarefa ou momento específico e alguns deles são mesmo divididos em "sub" eventos separados. Normalmente estes eventos são executados pelo GameMaker Studio 2 quando ele detecta um gatilho dentro do jogo, ou cada tick do jogo (como é o caso da etapa e dos eventos de sorteio), mas você também pode aplicar um evento a uma instância na sala atual de dentro de um pedaço de código.

Existem as seguintes funções que lidam com a geração de eventos:

  • evento_erecido
  • event_perform
  • event_perform_object
  • event_user

Há também uma série de variáveis que podem ser usadas apenas para leitura com as funções listadas acima e que são particularmente úteis para depuração:

  • evento_ação
  • número_do_evento
  • event_object
  • tipo_de_evento

Para mais informações sobre eventos, consulte a seção Eventos Objeto.
