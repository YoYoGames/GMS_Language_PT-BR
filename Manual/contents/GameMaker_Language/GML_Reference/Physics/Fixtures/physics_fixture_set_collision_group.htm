grupo_fixo_físico_set_collision

Uma vez criada sua fixação, você pode defini-la como fazendo parte de um grupo de colisão. Isto significa que você pode dizer a instâncias com esse dispositivo para sempre colidir ou nunca colidir com outra instância que tenha dispositivos ligados a elas com o mesmo grupo de colisão. Os valores que podem ser atribuídos são baseados em números positivos ou negativos, sendo 0 o valor padrão (o que significa que a instância com esta configuração de fixação é ignorada a menos que um evento de colisão seja criado para ela). Aqui está uma imagem para ajudar a explicar:

Como você pode ver, a definição de grupos de colisão pode ser muito útil para criar grupos complexos corpos físicos que interagem com o mundo físico ao seu redor, mas não uns com os outros. Entretanto, deve-se notar que devido à complexidade de calcular colisões com base em grupos como este, é recomendável que se tente manter o número de grupos em uso em qualquer sala a um mínimo absoluto.

Sintaxe:

physics_fixture_set_collision_group(fixture, group)

Argumento Descrição
fixação a fixação que deve ser amarrada
grupo o grupo ao qual esta fixação é atribuída (ou um valor positivo, 0, ou um valor negativo)

Devoluções:

N/A

Exemplo:

physics_fixture_set_collision_group(fix_Wheel, -1);

O código acima definirá a fixação indexada na variável "fix_Wheel" para fazer parte do grupo "-1".
