Esta seção é para aqueles usuários que tiveram acesso ao Steam API para publicar seu jogo naquela plataforma e que querem usar as possibilidades que o Steam Workshop e a Comunidade lhe dão para adicionar e gerar conteúdo de usuário em seu projetos. A forma mais simples de conteúdo gerado pelo usuário é a capacidade de tirar e compartilhar capturas de tela, o que é facilitado utilizando as duas funções a seguir:

  • steam_is_screenshot_requested
  • steam_send_screenshot

O restante das funções desta página estão relacionadas ao uso do Steam Worksop e algumas delas requerem o Steam App ID para seu projeto, bem como um User ID para a pessoa que joga o jogo ou seu nome do usuário, que pode ser encontrado usando as seguintes funções:

  • steam_get_app_id
  • steam_get_user_account_id
  • steam_get_user_persona_name

Antes de usar qualquer uma das funções integradas para o Steam UGC(User Generated Content) API você precisa ter configurado seu jogo corretamente a partir do painel de controle do Steam e você deve ter lido através do necessário documentação encontrada aqui:

  • Compartilhamento do conteúdo gerado pelo usuário

NOTA: Você precisa ter seu jogo aceito para a loja on-line Steam e ter acesso às áreas de desenvolvimento da documentação da API Steam.

Todos os itens UGC subscritos serão baixados automaticamente pelo cliente Steam, e você deve ter o código no Evento AssíncronoSteam para pegar isso e armazenar o ID do UGC que foi baixado para uso nas outras funções do UGC.

IMPORTANTE! As identificações UGC a vapor podem ser em grande número acima de 600.000.000. Isto significa que às vezes você pode precisar armazená-los como um fio em vez de tentar armazená-los como um valor real, especialmente se estiver trabalhando com amortecedores ou tentando escrever o valor para um arquivo de texto (já que isto o converterá para um formato padrão simplificado como "6.6624e+003", o que causará problemas de leitura de volta).

O fluxo de trabalho normal para colocar a UGC em seu jogo seria o seguinte:

  1. O usuário assinaria um item (seja de seu jogo usando steam_ugc_subscribe_item() ou do cliente/browser.
  2. Seu jogo "escutaria" para novas assinaturas com o código apropriado no Evento Steam Async.
  3. Quando você recebe um evento de assinatura assimétrica, você então verifica se o item está instalado (ou seja: baixado) com steam_ugc_get_item_install_info().
  4. Se o item não estiver instalado, você precisará de um cheque usando steam_ugc_get_item_update_info() para acompanhar o progresso do download.

Uma vez que seu conteúdo de usuário tenha sido criado e a oficina esteja disponível para download, as pessoas podem assiná-lo através do aplicativo Steam ou através do portal da Web. Entretanto, o GameMaker Studio 2 também inclui as seguintes funções para usar o Steam API para criar e cancelar assinaturas, assim como para obter informações sobre o que o usuário está assinando atualmente:

  • steam_ugc_subscribe_item
  • steam_ugc_unsubscribe_item
  • steam_ugc_num_subscribed_items
  • steam_ugc_get_subscribed_items
  • steam_ugc_get_item_install_info
  • steam_ugc_get_item_update_info
  • steam_ugc_request_item_detail

  • vapor_ugc_download
