Estruturas de dados

Nos jogos, muitas vezes você precisa armazenar informações de forma precisa e ordenada. Por exemplo, você pode precisar armazenar listas de itens que uma pessoa carrega ou pode querer armazenar uma grade de lugares que ainda precisam ser visitados. Agora, você pode usar as matrizes para isso, mas se você quiser fazer operações mais complicadas, como classificar os dados ou procurar por um item em particular, você precisa escrever grandes peças de código GML que podem ser lentas de executar, difíceis de depurar e uma verdadeira dor de escrever!

Para remediar isso, o GameMaker Studio 2 possui uma série de estruturas de dados embutidas que podem ser acessadas através de funções especializadas. Há seis tipos diferentes de estruturas de dados disponíveis, cada uma com seus próprios benefícios, dependendo do tipo de informações que você está procurando armazenar e como deseja manipulá-las posteriormente: pilhas, filas, listas, mapas, filas prioritárias e grades.

Essencialmente, todas as estruturas de dados funcionam da mesma maneira - Você cria uma estrutura de dados e armazena seu valor de índice em uma variável. Você então usa este índice para referenciar a estrutura de dados em todas as outras chamadas de função que podem realizar operações em isso. Finalmente, uma vez terminado, você destrói novamente a estrutura de dados para removê-la da memória. Você pode usar tantas estruturas quantas forem necessárias ao mesmo tempo, e todas as estruturas podem armazenar qualquer um dos tipos de dados disponíveis.

NOTA: Como acontece com todos os recursos dinâmicos, as estruturas de dados ocupam a memória e assim devem ser sempre destruídas quando não são mais necessárias para evitar vazamentos de memória, o que irá retardar e, eventualmente, quebrar seu jogo.

IMPORTANTE! Quando você cria uma estrutura de dados, o valor do índice para identificá-lo é um valor inteiro começando em 0. Isso significa que diferentes estruturas de dados podem ter o mesmo valor de índice, portanto, em caso de dúvida, você deve estar usando o ds_existe antes de acessá-los. Observe também que os índices são reutilizados, portanto um valor de índice de estrutura de dados destruído pode ser usado por um índice recém-criado posteriormente, e recomendamos que você defina qualquer variável que contenha um índice DS para -1 após ter destruído a estrutura de dados.

As informações sobre as diferentes estruturas de dados podem ser encontradas nas seções seguintes:

  • Grades
  • Listas
  • Mapas
  • Filas de espera prioritárias
  • Filas de espera
  • Pilha

Antes de utilizar estruturas de dados em seu jogo, você também deve estar ciente de que há certos momentos em que, devido a erros de arredondamento, você pode obter um resultado que não é o esperado. Isto pode ser resolvido alterando a precisão do DS usando o seguinte função:

  • ds_set_precision

Há também uma função especial para verificar se existe uma estrutura de dados de qualquer tipo:

  • ds_existe
