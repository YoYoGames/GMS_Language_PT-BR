Controle da fonte

Source Control Management (SCM) é o nome dado ao método de trabalho com sub-versões ou backups de seus projetos através de um repositório e fonte local. Basicamente, uma solução SCM é um pacote de software independente que controla todos os aspectos de manutenção, mudança e comparação das versões de seu projeto enquanto você trabalha nele. Isto é especialmente útil para aqueles que trabalham em equipe e precisam ser capazes de controlar quem faz o quê e não se preocupar em perder dados ou fazendo mudanças que podem precisar ser desfeitas posteriormente, mas os indivíduos também podem se beneficiar deste sistema poderoso, porém flexível.

Há várias ferramentas SCM disponíveis para você, e o GameMaker Studio 2 vem com um plugin Git incluído que pode ser usado imediatamente, e você não tem necessidade de instalar nenhum pacote extra, já que está tudo empacotado com o próprio plugin. Abaixo você pode encontrar links para um pequeno tutorial sobre como configurar este plugin e usar as ferramentas SCM com um projeto.

Quando você tiver ativado o Controle de Fonte para qualquer projeto (veja a seção Configurando o Plugin de Controle de Fonte acima para detalhes), o GameMaker Studio 2 mostrará o seguinte menu de contexto em o topo da IDE:

Aqui esboçamos cada uma das opções disponíveis (a maioria delas é explicada com mais profundidade nas seções de tutorial acima):

  • Criar Repositório de Projetos: Esta opção permite que você crie um repositório local no mesmo diretório que seus arquivos de projeto.
  • Projeto de Importação para Repositório: Esta opção permite a importação de um projeto para um repositório externo.
  • Repositório de Clones: Esta opção lhe permitirá clonar um repositório de uma fonte para um novo destino.
  • Comprometer mudanças: Com esta opção você pode encenar arquivos alterados em seu projeto e depois submetê-los ao repositório.
  • Mudanças de impulso: Após fazer um commit ou uma fusão, esta opção é usada para empurrar as mudanças para o repositório principal.
  • Mudanças de Puxar: Com esta opção, você pode atualizar o repositório local puxando os arquivos alterados do repositório principal.
  • Ver Histórico: Isto abrirá a janela do histórico onde você poderá visualizar todo o histórico de versões do projeto e optar por reverter caminhos de arquivos específicos ou revisões inteiras.
  • Mostrar Conflitos: Isto abrirá a janela Conflicts e listará quaisquer arquivos conflitantes que possam existir no projeto, permitindo que você lide com eles através da IDE GameMaker Studio 2 ou usando uma Ferramenta de Integração específica.
