Preferências de idiomas

As Preferências de Linguagem são usadas para definir como algumas das características da linguagem para programação serão exibidas na IDE do GameMaker Studio 2. As opções para GML são as seguintes:

  • Mostrar os tipos de argumentos na barra de status: Esta opção se refere a quando você estiver usando seus próprios scripts personalizados e tiver usado a notação JSDoc para identificar os diferentes argumentos, etc... (veja aqui para mais informações). Quando esta opção for marcada, e você tiver especificado os tipos de argumentos no script, então eles serão mostrados na barra de status na parte inferior do editor, incluída em {}. Isto está desativado por padrão.
  • Exibir descrições de argumentos na barra de status: Se isto for verificado, qualquer descrição que você tenha dado a seus scripts personalizados usando a notação JSDoc será mostrada na barra de status na parte inferior da janela do editor. Isto está desativado por padrão.
  • Mostrar a ortografia americana para as funções do sistema: Por padrão o GameMaker Studio 2 usa a ortografia inglesa GB para funções, por exemplo draw_text_colour(), porém isso pode não ser exatamente aquilo a que você está acostumado e assim você pode mudar para as ortografias dos EUA usando esta opção, de modo que as funções se tornem draw_text_color(), por exemplo. Isto é desligado por padrão.
  • Número máximo de erros de sintaxe a exibir: Este valor define o número máximo de erros de sintaxe que serão exibidos a qualquer momento. Se você achar que a IDE está atrasada, então você pode tentar reduzir este valor. Este valor é definido como 50 por padrão.
  • Habilitar avisos de referência variáveis: Quando ativado, isto marcará qualquer problema potencial com variáveis, como quando uma foi definida em algum lugar, mas não é realmente utilizada no código. Quando desativadas, estas advertências não serão mais mostrado. O padrão é que isto esteja ativado.
