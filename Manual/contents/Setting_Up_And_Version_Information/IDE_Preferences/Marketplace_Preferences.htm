Preferências demercado

As Preferências deMercado são usadas para configurar o Certificado deMercado para carregar ou comprar ativos. O formulário do certificado consiste nas seguintes seções para quando você já tiver criado um certificado (veja mais abaixo nesta página para detalhes sobre como criar um certificado):

  • Arquivo de Certificado: Esta seção é onde você especifica o arquivo *.pem que o Marketplace utiliza para a certificação de bens. Se você tiver um certificado criado anteriormente, então você pode clicar no navegador de arquivos aqui e adicioná-lo ao fórum, mas se você não tiver um certificado e desejar fazer um, então você deve omitir qualquer informação aqui e em vez disso clicar no botão Criar Novo Certificado (explicado abaixo).
  • Senha do certificado: Aqui você deve inserir a senha do certificado para o arquivo selecionado acima. A senha é criada quando você cria pela primeira vez o arquivo *.pem (veja Criar um novo certificado, abaixo) e será irreversivelmente vinculado ao arquivo de certificado criado.
  • Verificar: Uma vez preenchidos os campos acima, você deve clicar neste botão para garantir que a senha e a seleção do arquivo estejam corretas e funcionem em conjunto.

Confirmar Diálogos

Aqui você pode selecionar o comportamento dos diálogos de confirmação ao trabalhar com um recurso do Navegador de Ativos. As opções disponíveis são:

  • Resposta automática à exclusão de conteúdo offline: Quando você exclui um bem de mercado de sua biblioteca sem uma conexão web, você normalmente recebe um aviso perguntando se tem certeza de que deseja prosseguir. Ao definir esta opção como "OK". a mensagem será suprimida como se você tivesse clicado no botão "OK" para prosseguir. O valor padrão é "Mostrar mensagem".

Criar um novo certificado

Se você não tiver feito anteriormente um arquivo de certificado para usar o Marketplace, então você deve clicar no botão Create New Certificate (Criar novo certificado ) no final deste formulário. Quando você fizer isso, uma nova janela se abrirá solicitando que você insira as seguintes informações:

  • Nome da editora: Este é o nome que será usado como o nome da Editora para o bem. Dentro do painel do Marketplace você pode criar diferentes editoras se precisar e cada uma deve ter o seu próprio *.pem correspondente arquivo de certificado.
  • Senha do certificado: Aqui você deve fornecer uma senha única para o arquivo do certificado a ser criado. Tome cuidado com o que você escolher, pois não poderá mudá-lo mais tarde e se você não tiver a senha correta para o arquivo de certificado em as seções acima não poderão atualizar nenhum ativo existente no Marketplace (nem carregar novos ativos).

Com esses detalhes fornecidos, você pode então clicar no botão Criar Certificado para gerar o arquivo *.pem. Você será solicitado a dar um local de salvamento e um nome para o arquivo, e você deve tomar nota disso para referência futura. Uma vez feito isso, o novo arquivo e a senha serão automaticamente adicionado aos campos do formulário listados acima e você pode clicar no botão Verificar para testar se o certificado foi gerado corretamente.

OBSERVAÇÃO: Recomendamos que você tire uma foto da tela das configurações reais, bem como faça uma cópia do arquivo *.pem . Ambos devem ser salvos juntos em um local seguro para referência futura, ou no caso de ser necessário reinstalar o GameMaker Studio 2 em algum momento. Você não pode atualizar ativos que já foram carregados, a menos que o pacote atualizado seja assinado usando o mesmo certificado com o qual o original foi criado.
