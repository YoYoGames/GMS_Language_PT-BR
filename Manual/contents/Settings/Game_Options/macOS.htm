Opções de jogomacOS

Esta seção descreve as diferentes opções disponíveis para você que controlam como seus projetos de jogo macOS serão compilados. As diferentes seções são:

Geral

Na seção Informações do produto , a primeira coisa a fazer é preencher o Nome de exibição de seu jogo em seguida, preencha seu número de versão, ID do aplicativo, arquivo de saída do aplicativo e os detalhes de Copyright. O App Output é o local para o qual seus arquivos de instalação do jogo vão no desenvolvimento de seu Mac. máquina. Quando você cria seu .app.zip, esse arquivo pode ir a qualquer lugar (o local que você especificar ao fazer o diálogo de gravação não é a única cópia feita daquele .app.zip), mas o .pkg e outros arquivos são deixados nesta pasta, permitindo que você teste instalações locais via Terminal de forma a simular a compra na Loja Mac, etc...

Aqui você também precisa dar ao Team Identifier que você deseja usar para assinar o aplicativo final que o GameMaker Studio 2 cria para você. Colocá-lo aqui irá substituir o ID da Equipe que você forneceu nas PreferênciasMacOS. Após o Identificador de Equipe você também tem a opção de fornecer o Identificador de Assinatura, que é exigido pela Apple para todas as aplicações que não sejam de loja (como jogos a vapor, por exemplo).

A próxima opção é para habilitar Vapor em seu jogo. Se você ativar esta opção, certifique-se de ter o SteamworksSDK instalado (veja aqui), e que você adicionou a ID do aplicativoSteam nas Opções Gerais do jogo.

Finalmente, você tem a opção de Desativar o sistema de arquivo sandbox. Ao verificar isso, você desativará o GameMaker Studio 2 sandboxing para salvar e carregar arquivos, permitindo que seus jogos acessem arquivos de qualquer lugar do sistema que esteja rodando o jogo. Esta opção é desmarcada por padrão, pois recomendamos que você mantenha o sandbox do GameMaker Studio 2 para arquivos em todos os sistemas e, quando marcada, algumas áreas de salvamento/carregamento podem ainda não estar disponíveis, dependendo das permissões de nível de sistema operacional. Para maiores informações na caixa de areia, consulte as páginas do Sistema de Arquivos do GameMaker Studio 2.

Gráficos

As opções gráficas são aquelas que você deve configurar para determinar como seu jogo irá utilizar a placa gráfica de seu Mac alvo. As seguintes opções estão incluídas para que você possa modificar:

  • Permitir Menu e Dock em tela cheia: Quando marcada, esta opção mostrará o menu e a doca do SO se o jogo estiver no modo de tela cheia. Esta opção está desligada por padrão.
  • Cursor de exibição: Quando isto for verificado, o cursor normal do MacOS será mostrado, e desmarcá-lo significará que nenhum cursor será mostrado a menos que você tenha criado um em seu código de jogo. Isto está desativado por padrão.
  • Iniciar tela cheia: Se isto for verificado, o jogo começará no modo de tela cheia, caso contrário, começará em tela cheia. Isto é desligado por padrão.
  • Permitir a troca de tela cheia: Com isso, o usuário pode mudar de tela cheia para janela e voltar usando os atalhos macOS padrão. Isto é desligado por padrão.
  • Interpolar as cores entre os pixels: Ativa a interpolação linear, que basicamente "suaviza" os pixels. Para gráficos com pixels nítidos, deve ser desligado, mas se você tiver misturas alfa agradáveis e gráficos alisados é melhor deixá-lo ligado. Isto está desligado por padrão.
  • Use a sincronização para evitar rasgos: Esta sincronização ativa ou desativa a sincronização v-sync. Note que se você tiver um jogo com uma velocidade de sala de 120 e o jogador tiver um monitor com uma taxa de atualização de 60, ligando esta opção, sua velocidade de jogo também será fixada em 60. Esta opção está desligada por padrão.
  • Permitir o redimensionamento da janela: A verificação disto permite ao usuário mudar o tamanho da janela do jogo (a opção Janela sem Fronteiras precisa estar desligada para que isto funcione). Esta opção está desativada por padrão.
  • Habilitar a Retina: Ao verificar isso, o buffer traseiro será ajustado para a resolução mais alta (real) quando o jogo for executado em um monitor habilitado para retina, enquanto que ao desmarcá-lo, o buffer traseiro será ajustado para a resolução aparente (mais baixa). O que isto significa é que a janela inicial do jogo será ajustada para o dobro da largura e altura que a sala/ porta de visualização que a primeira sala estiver ajustada. No entanto, isto NÃO escalona a superfície de aplicação para se adequar, e assim, se você quiser que seu jogo faça a a maior parte da tela da retina você também deve definir a superfície de aplicação para ser a mesma que o tamanho da janela (note que isso duplicará os pixels ao longo da largura e da altura, portanto só habilite isso e escale a superfície de aplicação se seu jogo for beneficiado da resolução aumentada).

    Esta opção está desligada por padrão, e habilitá-la não mostrará nenhuma diferença quando você executar ou depurar o jogo usando VM, já que o corredor é pré-construído nesse caso. Ele só funcionará se você construir um executável através de VM, ou usar o YYC para testes ou construindo o jogo, pois isso reconstruirá o corredor com suas opções de jogo personalizadas.
  • Escalonamento: Aqui você pode optar por manter a relação de aspecto (assim uma sala de 4:3 será "encaixotada" em um 16:9) ou escalar completamente (esticando a imagem para caber na tela inteira).

    ADVERTÊNCIA! Adesativação da superfície de aplicação desabilitará todas as opções de escalonamento definidas nas Opções de Jogo do Windows até que ela seja ligada novamente. Veja a superfície de aplicação para mais detalhes.

Finalmente, há a opção de definir o tamanho da página de textura. O tamanho padrão (e o mais compatível) é 2048x2048, mas você pode escolher entre 256x256 até 8192x8192. Lá é também um botão marcado Pré-visualização que gerará as páginas de textura para esta plataforma e depois abrirá uma janela para que você possa ver como elas são. Isto pode ser muito útil se você desejar ver como são as páginas de textura estruturada e para evitar ter páginas de textura maiores (ou menores) do que o necessário. Para maiores informações sobre páginas de textura, por favor, veja aqui.

NOTA: Esteja ciente de que quanto maior o tamanho da página de textura, menos compatível seu jogo estará nos PCs com especificações mais baixas.

Imagens

A seção de imagens é onde você fornece as imagens que seu jogo requer. Para Mac, você precisa fornecer um arquivo Icon (que deve estar no formato .png e 1024x1024px) e uma tela Splash (que pode ser no formato .png, .bmp, .jpg ou .gif ). A tela splash será mostrada enquanto o jogo carrega.

Para os pacotes DMG, você pode fornecer uma imagem de fundo do instalador que aparecerá como fundo do instalador DMG.

Vale notar que o GameMaker Studio 2 possui uma ferramenta Gerador de Imagens de Projeto que pode ser usada para criar automaticamente todas as imagens necessárias para todas as diferentes plataformas alvo de seu jogo está sendo compilado para. Se você usar esta ferramenta, você deve revisar as imagens criadas para garantir que elas sejam o que você precisa.

Embalagem

Se você deseja que o jogo acabado esteja pronto para a App Store Ready, então você precisa verificar a opção Build for Mac App Store, mas esteja ciente de que isto só funcionará se você for um Desenvolvedor registrado e tiver os certificados necessários. Sob esta são as permissões do aplicativo que você só deve verificar se são verdadeiras, pois a Apple pode rejeitar seu aplicativo se eles são verificados quando não são necessários ou vice-versa. Estas opções simplesmente permitem que seu jogo utilize as funções http_ e url_. É importante notar que se você deseja apoiar os GamePads em seu jogo, então a opção de criar um pacote pronto para a App Store deve estar desligada.

Finalmente, você precisa selecionar sua categoria de aplicação (para mais informações, veja aqui).

Social

Esta opção é para usar o Apple Sign In Extension no Mac. Você pode obter esta extensão no Jogos YoYo Mercado, e a própria extensão contém instruções completas de uso. Se você não estiver usando esta extensão, então não deve assinalar esta opção.
