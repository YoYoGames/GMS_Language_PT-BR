Criando um Tutorial de Mercado

Criar um tutorial é semelhante à criação de um pacote de ativos Marketplace regular, somente agora você tem que adicionar em arquivos específicos incluídos, bem como incluir uma página inicial de tutorial (e uma página de vídeo opcional). O espaço de trabalho do Tutorial terá o seguinte aspecto:Como explicado acima para os ativos gerais do Marketplace, você precisa selecione uma editora e depois dê a seu tutorial um nome e uma identificação única (formato de URL reversa), e depois no menu suspenso "type" selecione Tutorial. Isto então lhe dará opções adicionais para adicionar um arquivo HTML base para iniciar o tutorial como bem como um arquivo (opcional) de vídeo e URL. O arquivo de vídeo deve usar o modelo fornecido pelo Marketplace - assim como o resto do tutorial - que você pode obter aqui, e quando o usuário carrega o tutorial, o vídeo será carregado em um quadro separado dentro da IDE. Observe que as páginas fornecidas para a página inicial e a página de vídeo são caminhos relativos dentro dos Arquivos Incluídos, e você pode abrir um explorador para escolher algo diferente dos padrões com os quais o GameMaker Studio 2 preenche os campos.

A criação de seu tutorial exigirá que você adicione Arquivos Incluídos em seu projeto tutorial, e estes devem ser adicionados de uma forma muito específica para que a IDE possa detectar corretamente quais arquivos devem ser usado para o tutorial. Temos, portanto, duas pastas principais:

  • TutorialConteúdo: Todos os arquivos de seu tutorial HTML, o CSS para estilizá-los, e as imagens necessárias para eles (não as imagens necessárias para os recursos, mas as que devem ser exibidas dentro do tutorial), bem como o modelo de vídeo, se necessário, vá aqui. Você pode ter também subpastas se desejar dividir o tutorial em capítulos, por exemplo, mas você deve ter pelo menos um arquivo base HTML para ser usada como página inicial para o tutorial.
  • TutorialRecursos: Esta pasta tem mais duas subpastas para Imagens e Ativos Sonoros . Se seu tutorial exige que o usuário adicione um sprite (por exemplo) e você deseja incluí-lo no tutorial então você colocaria o recurso de imagem na pasta Imagens. Estas pastas são importadas para o projeto tutorial e quando o usuário abre o editor sprite e vai adicionar uma imagem, o caminho padrão para a janela do explorador será para este local, tornando mais fácil para o usuário encontrar os bens necessários.

Para que seu tutorial funcione dentro da IDE você deve ter esta estrutura de pastas, bem como ter uma página inicial HTML com um arquivo CSS para estilização. Há uma série de links especiais que podem ser usados dentro de seus projetos tutoriais que saltarão para partes de um vídeo, ou abrir o manual em uma seção ou função específica, e todos estes são dados nos modelos oficiais do Tutorial do Marketplace.

IMPORTANTE! Sugerimos fortemente que você baixe os Modelos de Tutorial do Marketplace antes de tentar criar seus próprios tutoriais. Eles contêm exemplos de todas as partes importantes do HTML, bem como CSS e imagens para recriar botões e menus dentro do tutorial, deixando você livre para se preocupar com o conteúdo em vez de como exibi-lo.

Uma vez que você tenha configurado os arquivos incluídos e fornecido as informações necessárias, você pode clicar em Ok e o tutorial será carregado no Marketplace, abrindo um espaço de trabalho na listagem de ativos quando terminado para que você possa editá-lo e adicionar ícones, texto, screenshots, etc... Uma vez publicado o tutorial, os usuários poderão encontrá-lo dentro da seção Tutoriais da Página Inicial, onde pode ser baixado e instalado.
