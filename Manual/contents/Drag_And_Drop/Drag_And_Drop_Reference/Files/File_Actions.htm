Biblioteca de Ação de Arquivos

As Ações de Arquivos tratam de dois tipos diferentes de arquivos - arquivos tampão e arquivos Ini - e também têm certas ações genéricas de arquivos para renomear ou copiar arquivos existentes.

Osarquivos de buffer são criados salvando os dados de um buffer que você criou anteriormente e estes dados podem então ser carregados novamente em um buffer a qualquer momento no futuro. Você pode encontrar mais informações sobre os buffers aqui: Ações de buffer.

Osarquivos Ini são arquivos pequenos, leves e compatíveis com a maioria das plataformas. São ideais para armazenar pequenas peças de informação, como preferências de interface, pontuações locais altas, dados de nível etc... e são muito fáceis de usar. Os arquivos Ini não para usar estas ações, e você pode ler a partir de um arquivo Ini inexistente e simplesmente receberá um valor de retorno padrão (que você especifica), entretanto recomendamos que você crie pelo menos uma "base ini para abertura e modificação antes de usar as ações. Este arquivo ini base pode ser criado simplesmente chamando a ação Open Ini File seguido pelo Close Ini File, uma vez que o fechamento o arquivo será gravado no disco, ou você pode incluir um no Arquivo Incluído do Navegador de Ativos. Se você estiver usando um arquivo incluído no Navegador de Ativos como sua base Ini, você também deve ler a seção do manual sobre como funciona o Sistema de Arquivo.

NOTA: Para jogos com localizações, é muito importante que as letras acentuadas possam ser lidas a partir de arquivos externos. Isto significa que você deve criar primeiro o arquivo ini no formato UTF8 e depois adicioná-lo ao GameMaker Studio 2 como um arquivo incluído. para que ele seja exportado na execução do jogo e utilizado no lugar do arquivo ini no formato padrão ANSI que é criado pelas funções ini quando nenhuma foi fornecida anteriormente. Desta forma, você pode ler e escrever corretamente com todos os sotaques e não cartas sendo mantidas.
