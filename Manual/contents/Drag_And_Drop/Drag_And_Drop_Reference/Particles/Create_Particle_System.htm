Criar Sistema de Partículas

Esta ação inicializará o sistema de partículas pronto para você criar e usar tipos de partículas e emissores com ele. Você dá o nome da camada como um fio - por exemplo, "Camada de Partículas" (que deve existir no editor da sala) - e então sinaliza o como sendo Persistente ou não. Se o sistema for marcado como Persistente, ele permanecerá ativo em todas as salas após ter sido criado, e lhe será atribuída uma camada gerenciada se a camada inicial não existir nas salas subseqüentes. Se a persistência não for assinalada, então o sistema será automaticamente destruído no final da sala em que foi criado, assim como quaisquer emissores associados ao sistema. Se você desejar remover manualmente o sistema da memória, você precisa usar a ação Destroy Particle System (Sistema de Partículas Destroy).

Você também precisa fornecer uma variável de destino que manterá o valor único de identificação do sistema de partículas. Este valor de identificação é o que será usado em outras funções para definir propriedades do sistema ou para dizer ao GameMaker Studio 2 qual sistema usar ao criar partículas. Você pode escolher marcar esta variável como sendo uma variável local temporária, mas isso significará que ela será descartada no final do script de ação ou evento, significando que o sistema ainda existirá mas deixará você sem nenhum meio de acessá-la, portanto cuidado devem ser tomadas quando se usa isto (em geral os sistemas nunca precisarão ser armazenados em uma variável local temporária, mas para certas coisas, como o desenho de partículas para uma superfície, pode ser necessário).

NOTA: Um sistema de partículas persistentes, uma vez criado, permanece residente na memória do sistema durante a duração do jogo, e assim, se você precisar recriá-lo a qualquer momento (por exemplo, se você chamar a ação Reiniciar Jogo ), então você deve destruir o sistema atual primeiro, caso contrário você terá um vazamento de memória. Um vazamento de memória acontece quando um recurso é adicionado à memória e depois "de-referenciado", significando que o GameMaker Studio 2 não tem como interagir com esse recurso não mais. Isto leva a que a memória fique entupida por recursos inacessíveis que, por sua vez, podem causar problemas com seu jogo. Para destruir um sistema de partículas persistentes quando não for mais necessário, use a ação Destroy Particle System (Destruir sistema de partículas).
