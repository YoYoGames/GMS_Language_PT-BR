Obter variável de instância

Com esta ação você pode obter qualquer uma das construído em variáveis. Todas as instâncias de todos os objetos têm uma série de variáveis já criadas dentro deles para conter informações essenciais como a posição x/y atual, a velocidade, a direção, a visibilidade e muitas outras coisas. As variáveis de instância que você pode obter os valores de são:

  • X Coordenada - A posição ao longo do eixo x na sala.
  • Coordenada Y - A posição ao longo do eixo y na sala.
  • Direção - A direção de movimento para a instância (o padrão é 0°, onde a direita é 0°, para cima é 90°, esquerda é 180° e para baixo é 270°).
  • Velocidade - A velocidade (em pixels por frame de jogo) em que a instância está se movendo.
  • Velocidade horizontal - A velocidade horizontal (em pixels por frame de jogo) em que a instância está se movendo. Uma velocidade negativa significa que ela está se movendo esquerda, uma velocidade positiva é a direita e o padrão é 0.
  • Velocidade vertical - A velocidade vertical (em pixels por frame de jogo) em que a instância está se movendo. Uma velocidade negativa significa que ela está se movendo para cima, uma velocidade positiva é baixa e o padrão é 0.
  • Visível - A variável de alternância de visibilidade (o padrão é verdadeiro, a menos que definido de outra forma no Editor de Objetos, definido como falso para torná-lo invisível). Note que uma instância invisível ainda executa todos os seus eventos , exceto o Evento de Sorteio.
  • Persistente - A variável toggle de persistência (o padrão é falso, a menos que definido de outra forma no Editor de Objetos), se esforçar para ser persistente). Isto fará com que uma instância "persista" de sala para sala, ou seja: a instância não será limpa na ponta da sala, mas será transportada para a sala seguinte. Note que quando uma instância é persistente não chamará seu evento Criar ou Destruir, mas chamará os eventos Limpeza, Fim de Sala e Início de Sala.
  • Camada - A camada para a qual a instância foi designada.
  • Profundidade - O valor de profundidade atribuído à camada.
  • Sprite - O recurso sprite designado para a instância.
  • Índice de imagem - O índice de imagem do duende atribuído. Este valor define a moldura do duende atribuído à instância, e se o valor for maior do que o número de quadros na animação, ele "loopará" de tal forma que um duende com 5 quadros, se definido para o quadro 7, realmente mostrará o quadro 2.
  • Ângulo de imagem - O ângulo em que a instância deve ser definida (o padrão é 0°, onde a direita é 0°, acima é 90°, esquerda é 180° e abaixo é de 270°).
  • Imagem Alfa - O valor alfa para a instância (padrão é 1) de 0 a 1, onde 0 é totalmente transparente e 1 é totalmente opaco.
  • Image Blend - A cor a ser misturada com o sprite de instância (o padrão é branco).
  • Velocidade da imagem - A velocidade da animação, por exemplo.
  • Em Sequência - Esta variável retornará true se a instância estiver atualmente incluída em uma seqüência, e false caso contrário.
  • Escala horizontal - A escala horizontal na qual a instância sprite deve ser desenhada (isto também afetará a instância delimitadora caixa e colisões). Uma escala de 1 indica que não há escala (1:1), valores menores serão escalados para baixo (0,5, por exemplo, será metade da largura do duende), valores maiores serão escalados para cima e valores negativos espelharão o duende e o escalarão, a menos que o valor utilizado é exatamente -1 (neste caso o duende é apenas espelhado ao longo do eixo y, sem escala).
  • Escala vertical - A escala vertical na qual a instância sprite deve ser desenhada (isto também afetará a instância delimitadora caixa e colisões). Uma escala de 1 indica que não há escala (1:1), os valores menores serão escalados para baixo (0,5, por exemplo, será metade da altura do duende), os valores maiores serão escalados para cima e os valores negativos serão escalados para cima , a menos que o valor utilizado é exatamente -1 (neste caso, o duende é simplesmente virado ao longo do eixo y, sem escala).
  • Máscara - O índice da máscara, por exemplo. O valor padrão é -1 (o que significa que a instância usará a máscara de colisão para a atualmente atribuído ao índice sprite), mas se definido para qualquer outro recurso sprite, será usada a máscara de colisão para aquela instância em vez do sprite atribuído ao índice sprite.
  • Fricção - O atrito para usar cada frame do jogo para retardar a instância. Este valor será simplesmente deduzido do atual valor da velocidade de cada quadro de jogo (ou adicionado se o valor da velocidade for negativo).
  • Gravidade - A gravidade aplicará uma força constante na direção da gravidade (ver abaixo) da instância, influenciando sua velocidade (o padrão é 0). Note que a gravidade é uma força cumulativa e irá acelerar o objeto se você optar por não limitar a velocidade final.
  • Direção da gravidade - Pode ser usado para definir a direção do movimento quando a gravidade é maior que 0 (padrão é 270). Note que as instruções no GameMaker Studio 2 são calculadas como 0° sendo a direita, 90° sendo para cima, 180° sendo para a esquerda e 270° sendo para baixo.
  • Pontuação - Defina a variável de pontuação.
  • Vidas - Defina a variável vidas.
  • Saúde - Defina a variável saúde.

Uma vez selecionada a variável embutida a ser obtida, você então define a variável alvo para manter o valor retornado. Se você marcar esta variável como uma variável temporária, ela será criada para você na ação e estará disponível apenas até o final do evento ou roteiro de ação. Você também pode recuperar os valores de variáveis adicionais na mesma ação, clicando no ícone de mais ao lado da ação, e selecionando outra variável de instância e dando uma variável de acompanhamento para armazenar o valor retornado.
