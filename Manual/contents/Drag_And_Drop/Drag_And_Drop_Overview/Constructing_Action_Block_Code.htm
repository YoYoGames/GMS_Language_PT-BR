Construindo o Código de Bloco de Ação

Para adicionar comportamentos aos objetos, você pode construir seu código usando as ações das diferentes bibliotecas disponíveis na Caixa de Ferramentas de Ação. Para começar, você precisará fazer um novo projeto de Drag and Drop e depois fazer um novo objeto ( você pode atribuir um sprite ao objeto também, se necessário). Em seu novo objeto, você pode começar a adicionar eventos, e nos eventos acrescente suas ações em código DnD™.

Note que quando você adiciona um novo evento, uma janela de "código" é aberta com uma aba para o evento em questão (veja a imagem acima), e agora você pode arrastar qualquer ação que desejar da Caixa de ferramentas à direita para o painel de blocos de ação da janela de código. Agora, embora seja verdade que você pode adicionar qualquer ação, isso não significa que todas elas funcionarão ou que o projeto irá compilar com eles. Algumas ações exigem pelo menos uma variável para funcionar, enquanto outras - como as ações do sorteio - só trabalho se usado em um evento específico. Como você sabe quais usar? Bem, geralmente é simplesmente uma questão de usar a lógica... se uma ação requer uma variável e ainda não a definimos, então não devemos usá-la até que acrescentemos uma ação para criar a variável.

Quando você arrasta uma ação da Caixa de Ferramentas para o espaço de trabalho principal do Bloco de Ação, ele se expandirá para mostrar osargumentos (parâmetros) disponíveis que você pode preencher e mudar para definir o comportamento. Variáveis-alvo

É importante notar que muitas ações oferecem uma variável "alvo", que pode ser sinalizada como "Temporária", por exemplo:

O que isso significa é que você pode fornecer uma variável que é a meta para manter o valor retornado pelo ação. No exemplo de ação acima, a ação retornará o volume do recurso de áudio que você selecionar, então você fornece uma variável alvo para manter este valor para que você possa referenciá-lo mais tarde.

Agora, esta variável alvo precisará ter sido declarada anteriormente ou usando a Assignar Variável (que cria uma Variável de Instância) ou Declare Variável Temporária (que Se você marcar a opção "Temp", então você pode simplesmente adicionar um nome de variável e a ação criará essa variável e a definirá para manter o valor de retorno para você (criando uma variável local temporária variável). Qualquer ação usada depois disto pode agora acessar o valor na variável temporária, mas somente dentro do mesmo Evento ou Roteiro. As variáveis temporárias estão disponíveis apenas dentro do escopo em que foram criadas. Para mais detalhes informações sobre variáveis e escopo de variáveis, por favor veja aqui.

Auto Completo

Ao trabalhar com Drag and Drop você terá que adicionar variáveis e expressões nos diferentes campos de entrada das ações. No entanto, ao fazer isso, muitas vezes você terá a janela do Auto Completo que aparecerá para ajudá-lo:

Esta janela irá listar todas as variáveis GML(GameMaker Language) construídas, constantes e funções, como assim como quaisquer recursos que contenham as letras iniciais do que você estava digitando. Ele pode ser usado para encontrar rapidamente o recurso ou variável que você deseja para a ação sem ter que digitar tudo em você mesmo. Por exemplo, se todos os seus quartos são prefixados "rm_" e, em seguida, digitando isso e aguardando um momento, aparecerá a janela completa do auto com todos os recursos começando com "rm_". Note que todas as variáveis embutidas que são mostrados na janela auto completa podem ser usados em qualquer lugar onde uma variável ou expressão é esperada em uma ação, assim como a maioria das funções GML.
