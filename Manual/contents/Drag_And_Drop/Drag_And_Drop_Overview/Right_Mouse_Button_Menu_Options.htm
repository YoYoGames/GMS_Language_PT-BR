Opções do Menu do Botão Direito do Rato

Ao trabalhar em seu código DnD™, você pode clicar com o botão direito do mouse na ação, no espaço de trabalho ou no texto da lista restrita à esquerda para abrir o menu do botão direito do mouse:

Este menu tem as seguintes opções:

  • Cortar / Copiar / Colar: Isto lhe permitirá cortar, copiar ou colar qualquer ação que seja selecionada no espaço de trabalho. Note que você pode usar + fora das ações e arrastar, ou usar / + em ações individuais, para selecionar mais de um e aplicar o comando a todos eles.
  • Comentar / Remover Comentário: Esta opção lhe permite adicionar e remover comentários de seus blocos de código DnD™. Quando você selecionar Comentar, o editor de texto será aberto à direita do bloco DnD™selecionado, e lá você poderá adicione qualquer texto necessário, seja um lembrete para mudar algo, ou um comentário útil para um tutorial, ou mesmo instruções para seus colegas de equipe:
    Uma vez adicionado o comentário, você pode clicar nele e arrastá-lo para posicioná-lo verticalmente e horizontalmente - embora somente à direita do nó pai - e colapsando o nó DnD™também colapsará o comentário e mostrará um resumo de seu conteúdo (caso em que não poderá ser reposicionado). Se o comentário não for mais necessário, então você pode ou apagar todos os o texto no comentário, ou use a opção Remover Comentário do menu. Note que quando você copia um nó, o comentário será copiado junto com ele, e se você converter o nó de DnD™para GML, então o comentário será preservado. e adicionado ao GML que é criado, como ilustrado na imagem abaixo:
    Como mencionado acima, um comentário só pode ser aplicado a um único nó, mas você pode expandi-lo para cobrir vários nós, simplesmente adicionando intervalos de linha para estendê-lo - embora isso possa exigir que você reposicione o comentário para evitar sobreposições com nós subseqüentes:
    Uma vez adicionado um comentário, você pode selecionar a opção "Comentário". novamente a partir do menu RMB ou você pode clicar duas vezes sobre ele para habilitar a edição.
  • Desativar / Habilitar: Estes comandos podem ser utilizados para desativar ou ativar a ação (ou ações) selecionada. Quando uma ação é desativada, ela ainda é visível no espaço de trabalho, mas não será executada quando você compilar o jogo. Neste maneira de adicionar/remover ações para testes e depuração. A imagem abaixo mostra algum código DnD™com ações desabilitadas:
    Você pode ver que as ações "If Score" e "Go To Next Room" foram desabilitadas - foram acinzentados, visualmente - e por isso serão pulados quando o jogo for testado ou compilado.
  • Troca de ponto de parada: Este comando ativará ou desativará um ponto de parada para a ação selecionada. Um ponto de parada é simplesmente um ponto no código DnD™onde o depurador deve parar quando o jogo está sendo executado no modo Debug. Uma ação com um ponto de parada atribuído a ela será destacada, como mostrado na imagem abaixo (a ação "Se Saúde" foi definida como um ponto de parada):
  • Apagar todos os pontos de parada: Com este comando, você pode remover todos os pontos de parada do código DnD™.
  • Colapsar tudo / Expandir tudo: DnD™as ações podem ser colapsadas ou expandidas, onde uma ação colapsada simplesmente mostrará o nome da ação mas não qualquer um dos valores internos, maximizando assim o espaço disponível para o espaço de trabalho. Por exemplo:
    Como você pode ver, as ações ocupam muito menos espaço, e você ainda pode ver o que elas fazem a partir do curto descrição dada à direita. Você pode selecionar a opção "Expandir tudo" para desfazer isto para cada nó no espaço de trabalho.
  • Redefinir o layout: Isto irá redefinir o espaço de trabalho DnD™, ajustando o nível de zoom para 100%, focando na ação atualmente selecionada, e expandindo qualquer (e todas) as ações colapsadas também.
  • Pré-visualização ao vivo: Ao selecionar esta opção, você ativará ou desativará a janela de visualização de código DnD™para GML(GameMaker Language). Isto é explicado com mais detalhes na página Mudando DnD™ para Código.
  • Converter para GML: Isto irá converter todas as suas ações no evento atual para código GML(GameMaker Language). Tenha muito cuidado com isto, pois você não pode converter o GML de volta para as ações DnD™ das quais ele foi criado. Você pode converter o evento de volta para um evento DnD™de código, mas qualquer código será simplesmente envolto em uma ação de Código de Execução. Isto é explicado com mais detalhes na página Mudando DnD™ para Código.
  • Janela: Esta seção abre o seguinte sub-menu:
    Os itens de menu disponíveis aqui são:
    • Ir para Objeto: Isto vai direcionar o espaço de trabalho para o objeto que está chamando o evento
    • Evento Aberto Herdado: Esta opção só está disponível se o objeto sendo editado for filho de um objeto pai, e abrirá o evento sendo editado do objeto pai e abrirá o espaço de trabalho para ele.
    • Painel Único / 2 Colunas / 3 Colunas: Estas opções podem ser usadas para dividir o espaço de trabalho atual em 1, 2 ou 3 "painéis" diferentes. Cada painel é um espaço de trabalho separado que pode ter scripts ou eventos arrastados sobre eles para que você possa editar e comparar 2 ou mais blocos de código DnD™de cada vez. As duas opções a seguir só estão disponíveis quando o menu RMB é aberto em um nó DnD™:

  • Pan: Ao selecionar esta opção em qualquer nó, a área de trabalho irá girar para tocar aquele nó até o centro.
  • Mostrar Ajuda... : Selecionando isto abrirá o Manual em um novo espaço de trabalho na página relevante para o nó que foi selecionado. Você também pode obter os mesmos resultados usando no nó.
