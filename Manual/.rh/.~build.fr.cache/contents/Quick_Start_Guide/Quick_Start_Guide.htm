Guia de Início Rápido Bem-vindo ao Guia de Início Rápido! Esta seção do manual foi criada para familiarizá-lo com os aspectos mais importantes do GameMaker Studio 2 e lhe mostrará o básico para que você faça jogos o mais rápido possível. O guia abrange funcionalidade IDE essencial e também fornece uma breve introdução à programação, para que você possa aproveitar ao máximo o produto o mais rápido possível, quer você use a linguagem GameMaker(GML) ou a linguagem visual de scripting Drag e Drop™. Para ilustrar o fluxo de trabalho do GameMaker Studio 2 e fazer com que você faça algo, preparamos este guia para levá-lo do básico do que é GameMaker Studio 2 e do que é um programa, através da criação de recursos gerais e para cima de como codificar coisas básicas como movimento e pontuação. Fazer jogos com o GameMaker Studio 2 é, conceitualmente, um assunto bastante simples e direto, seguindo estas etapas gerais: Criar recursos de imagem ( sprites, conjuntos de azulejos, fontes) e adicioná-los ao Navegador de Ativos. Crie objetos ativos para representar as coisas que você deseja usar em seu jogo e atribua a eles os ativos de imagem necessários. Programar estes objetos para fazer coisas em resposta a eventos específicos. Adicione instâncias desses objetos em uma sala onde eles executarão seus programas. Nas páginas seguintes, entraremos nas especificidades de cada um dos itens listados acima, mas esse é o fluxo geral de ações necessárias para fazer um jogo. O importante aqui é observar que seus principais "blocos de construção" para o jogo serão sempre sprites, objetos e salas, embora haja uma série de ativos subsidiários que também podem ser usados para adicionar funcionalidade ou comportamentos extras a esses itens (como shaders, ou linhas do tempo). Agora, antes de continuarmos a explicar cada uma dessas coisas em mais detalhes, rapidamente esboçaremos como o GameMaker Studio 2 transformará os blocos de construção acima em seu jogo. Basicamente, tudo acontece no que é chamado de Loop do Jogo, que tem uma velocidade medida em quadros por segundo, e um "quadro" é chamado de Step in GameMaker . O frame padrão por segundo (ou FPS) é 60, o que significa que para cada segundo que passa no GameMaker O Studio 2 executará o loop do jogo 60 vezes, e cada passo (loop, frame) uma série de eventos ocorrerá e o jogo avançará. Para ajudar a entender isto, vamos imaginar que você tenha um personagem que queira mover-se pela sala de jogos. Você criaria um duende para esse personagem, depois criaria um objeto e lhe atribuiria o duende, e no objeto você adicionaria um evento chamado Step Event. Neste evento, você diria para se mover para a direita por 2 pixels. Você então colocaria uma instância desse objeto na sala de jogo, e quando você executasse o jogo, a instância começar a mover-se para a esquerda por 2 pixels a cada passo (o Evento de Passo é executado uma vez a cada passo do jogo), de modo que em 1 segundo a instância tenha movido 120 pixels para a direita. No exemplo acima, falamos em dizer ao objeto para se mover para a esquerda, mas como fazemos isso? Bem, isso é feito usando um programa que escrevemos usando um Drag e Drop™ (a linguagem visual de scripting) ou GML (a linguagem do Game Maker). Qual deles você usa depende de você e da maneira que você quer para trabalhar, mas ao usá-los, você está programando seu jogo.
