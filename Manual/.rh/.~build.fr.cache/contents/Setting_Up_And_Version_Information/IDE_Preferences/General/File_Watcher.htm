Preferências do Vigilante de Arquivos O Guardião de Arquivos é o nome dado à parte da IDE que observa os arquivos relacionados a um projeto para quaisquer alterações devidas a software externo (ou seja: edição de uma imagem em um editor externo) ou devido a alterações internas como a edição de um script ativo. Em geral, estas configurações não devem precisar ser editadas, mas em alguns sistemas - como aqueles que usam um SSD como unidade de gravação para os arquivos do projeto - o observador de arquivos pode estar informando sobre mudanças quando você faz coisas como simplesmente salvar o projeto ou sempre que você editar qualquer ativo do projeto, neste caso você pode precisar alterar estas configurações. As configurações disponíveis são: Habilitar o Vigilante de Arquivos: Isto pode ser usado para ativar ou desativar o sistema de Vigilância de Arquivos. Se isto for desativado, a IDE não mais captará nenhum arquivo alterado por fontes externas, portanto, deve-se ter cuidado ao desativar o sistema esta característica. O Vigilante de Arquivos é ativado por padrão. Defina a tolerância do Guardião de Arquivos no MS após salvar no IDE: Isto ajusta o tempo entre salvar um arquivo e a verificação de mudanças pelo vigilante de arquivos. Você ajustaria este valor se você estivesse salvando seu projeto e o Vigilante de Arquivos fosse Aparecendo erroneamente dizendo que algo havia mudado, nesse caso você aumentaria o atraso até que a mensagem não aparecesse mais.
