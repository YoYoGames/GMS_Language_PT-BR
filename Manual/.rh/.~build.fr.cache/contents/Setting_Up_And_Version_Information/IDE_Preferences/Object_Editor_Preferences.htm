Preferências do editor de objeto As Preferências do Object Editor são usadas para definir certas propriedades para a janela do Object Editor. A seção principal contém a seguinte opção: Clique para abrir Método de evento: Esta opção controla se você precisa realizar um clique duplo ou um único clique em um evento de objeto para abri-lo. Por padrão, isto é definido para dar um clique duplo. Eventos herdados alfa: Esta opção permite mudar o alfa com o qual qualquer evento hereditário será desenhado na lista de eventos do objeto. Por padrão, isto é definido como 0,5 (os valores devem estar entre 0 e 1). Formato do rótulo do evento: Aqui você pode ativar ou desativar os diferentes textos possíveis que podem ser aplicados à etiqueta do evento na lista de eventos do editor de objetos. Por padrão, a Descrição do evento é ativada apenas. Adicionar o conteúdo padrão do evento: Por padrão esta opção é ativada e tudo o que ela faz é adicionar um simples comentário no editor de código para todos os novos eventos, para lembrar que você pode adicionar seu código lá. Conteúdo do evento padrão: Aqui você pode editar os comentários que são adicionados por padrão na janela de código para novos eventos. A subcategoria a seguir também existe para o Editor de Objetos: Confirmar Diálogos Esta seção trata dos diálogos que podem ser mostrados durante a utilização do Object Editor, com a seguinte opção disponível: Resposta automática a DnD™ conversão para a linguagem Gamemaker: Você pode usar o botão direito do mouse dentro do espaço de trabalho Drag e Drop™para selecionar a opção de transformar todos os Drag e Drop™para esse evento em Código GML. A ação padrão aqui é mostrar uma mensagem de aviso perguntando se você quer continuar, mas você pode definir isto como "Ok" e o diálogo não será mostrado antes da conversão. Resposta automática ao exceder o limite de vértices físicos: Esta opção pode ser usada para controlar a mensagem pop up que é mostrada quando você excede o número de vértices permitido no editor de forma física. A ação padrão aqui é mostrar uma mensagem de aviso, mas você pode definir isto como "Ok" e o diálogo não será mostrado. Escolha automática na eliminação de eventos: Esta opção pode ser usada para controlar a mensagem pop up que é mostrada quando você exclui um evento na lista de eventos do editor de objetos. A ação padrão aqui é mostrar uma mensagem de aviso, mas você pode coloque isso em "Ok" e o diálogo não será mostrado.
