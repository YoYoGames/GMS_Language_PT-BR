Preferências de alimentaçãoem tempo de execução As Preferências de Alimentação em Tempo de Execução são usadas para exibir e alterar as preferências atualmente instaladas Tempo de execução bem como o tempo de execução em uso corrente. O GameMaker Studio 2 pode ter várias ferramentas diferentes de tempo de execução instaladas a qualquer momento e você pode definir qual delas usar a partir desta preferência. O ideal é que você queira ser usando o runtime mais atualizado, mas às vezes pode ter um problema, ou você pode ser obrigado a usar um mais antigo para comparabilidade, etc... e assim você pode mudar as ferramentas de runtime que o GameMaker Studio 2 usa para construir seu jogo aqui. A seção inicial desta preferência está relacionada a onde você puxa as ferramentas de tempo de execução e requer que você insira uma URL para um feed RSS. Este feed será então escaneado para atualizações das ferramentas de tempo de execução que podem então ser baixadas e instaladas. Esta seção mostrará a URL do feed Master runtime por padrão, mas você pode clicar no botão Adicionar para criar um novo Runtime Feed onde você dá o nome e a URL para o feed RSS, e então clique em Aplicar para dizer ao GameMaker Studio 2 para acessá-lo. Você precisará fechar e reabrir a janela de Preferências para que o novo feed seja mostrado. Uma vez criado um Feed de Tempo de Execução, ele será listado nesta seção como uma subcategoria, e você deverá sempre ter pelo menos um feed listado (por padrão, este seria o Feed Principal ). Independentemente do tempo de execução utilizado, todos eles terão as mesmas características, descritas abaixo usando a alimentação Master como exemplo: As preferências de Runtime Feed são usadas para selecionar, instalar e desinstalar as diferentes ferramentas de tempo de execução. Em o topo tem as ferramentas de tempo de execução em uso, a data em que foi criado e a localização em que está armazenado no disco rígido. Abaixo destes detalhes você pode encontrar uma lista de todos os tempos de execução disponíveis, e você pode escolher qualquer um deles lista para mudar para ser o tempo de execução "em execução" usado para compilar seus projetos. Para selecionar um tempo de execução diferente para usar, basta encontrá-lo na lista e depois clicar duas vezes sobre ele. Se o tempo de execução não tiver sido baixado e instalado, então você será solicitado a para fazer isso, e uma vez instalado será solicitado que você feche e reinicie a IDE para que o novo tempo de execução possa ser inicializado. Esta lista detalha todos os tempos de execução disponíveis, seu número de versão, a data em que foram liberados e - se forem instalados - o caminho para onde foram instalados. O lado direito também mostra os diferentes ícones de status que são os seguintes: Isto marca o tempo de execução atual que foi selecionado para ser utilizado na construção de seus projetos. Clicando aqui, será iniciado o download e a instalação do tempo de execução selecionado (uma barra de status será mostrada no canto superior direito da IDE). Clicando aqui, o tempo de execução selecionado será removido de seu disco rígido. Você pode atualizar a lista de tempo de execução a qualquer momento a partir do feed RSS dado clicando no botão de atualização , e você pode limpar seu disco rígido de tempos de execução antigos clicando o botão "Clean Runtimes Now ".
