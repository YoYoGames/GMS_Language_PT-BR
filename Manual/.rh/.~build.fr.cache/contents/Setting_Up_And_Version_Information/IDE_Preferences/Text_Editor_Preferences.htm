Preferências dos editores de texto As preferências dos editores de texto são usadas para definir como os diferentes editores de código/script são vistos e sentidos. A página principal tem as seguintes opções: Atraso na verificação da sintaxe (ms): O tempo de atraso (em milissegundos) antes da execução do verificador de sintaxe. Este valor padrão é de 2000ms. Atraso de conclusão do código (ms): O tempo de atraso (em milissegundos) Abrir scripts em editor de tela cheia: Ao marcar esta opção, os recursos de script serão abertos automaticamente em um novo espaço de trabalho. Se isto não for verificado, eles se abrirão em uma janela dentro do espaço de trabalho atual. Esta opção é desmarcada por padrão. Shaders abertos em editor de tela cheia: Ao marcar esta opção, os recursos de sombreamento serão abertos automaticamente em um novo espaço de trabalho. Se isto não for verificado, eles se abrirão em uma janela dentro do espaço de trabalho atual. Esta opção é desmarcada por padrão. Eventos de objeto aberto em editor de tela cheia: Ao marcar esta opção, os editores de código de eventos de objeto serão abertos automaticamente em um novo espaço de trabalho. Se isto não for verificado, eles se abrirão em uma janela acorrentada ao editor de eventos de objeto. Esta opção é desmarcado por padrão. Abrir scripts de eventos como abas na mesma janela: Ao adicionar código ou Drag and Drop a um evento, a janela de código é acorrentada à janela de evento, e quando esta opção for marcada, quaisquer outros eventos que forem abertos para edição serão adicionados para a mesma janela que as abas ao longo da parte superior. Desmarcando esta opção, cada evento será aberto como sua própria janela encadeada única. Esta opção é verificada por padrão. Abrir scripts vinculados como abas na mesma janela: Quando você tem vários scripts vinculados e os abre, por padrão eles serão agrupados na mesma janela que as abas. Desmarcando esta opção, eles serão forçados a abrir cada um deles em sua própria janela/espaço de trabalho. Esta opção é marcada por padrão. Editor de texto panorâmico em foco: Quando você abre um editor de texto a partir de um evento de objeto, selecionando esta opção, o espaço de trabalho será automaticamente panorâmico para trazer o editor para dentro dos limites. Esta opção está desligada por padrão. Largura da janela de código padrão: Defina a largura padrão (em pixels) para a janela de código a ser aberta. O valor padrão é 900. Altura padrão da janela de código: Defina a altura padrão (em pixels) para a janela de código a ser aberta. O valor padrão é 600. Largura mínima da janela de código: Defina a largura mínima (em pixels) permitida pela janela de código. O valor padrão é 300. Altura mínima da janela de código: Definir a altura mínima (em pixels) permitida pela janela de código. O valor padrão é 200. Além das opções acima, há também uma subcategoria para as preferências do editor de códigos: Editor de código O As preferências de cor são usadas para mudar a forma como o editor de código e roteiro exibe diferentes textos. Cada entrada tem um número de subentradas para permitir que você defina exatamente como cada pedaço de texto separado deve ficar, como mostrado no exemplo imagem acima. Em geral, você pode definir a cor, fonte, tamanho e estilo para cada uma das diferentes opções. Mais abaixo, você também pode definir um estilo mais geral para coisas como seleção de linhas de parênteses de código, etc... Sob as preferências de cor você tem as diferentes opções de estilo. Estas preferências são usadas para definir o comportamento do editor de código/script, com as seguintes opções disponíveis: Fique de olho: Se for verificado, isto irá manter a tecla de tabulação pressionada como caracteres de tabulação - ' \t ' - no editor, enquanto desmarcado significa que as abas são convertidas em espaços. O padrão está desligado. Tamanho da aba: O número de espaços de caracteres que ao pressionar a tecla "Tab" deve indentar o código. O padrão é 4. Mostrar o espaço para brancos: Mostrar " . " para mostrar o espaço em branco (ver imagem abaixo). O padrão está desligado. Mostrar números de linha: Mostrar os números das linhas à esquerda ou não. O default está ativado. Mostrar linhas de aba: Mostrar " .... " para mostrar o espaço das abas. O padrão está desligado. Habilitar o código de cores: Habilitar ou desabilitar a codificação por cores para o editor. Se desativado, o código não será estilizado, enquanto que quando ativado será estilizado usando a formatação dada acima na seção Cores. O padrão está ativado. Limite de linha de documento: Defina o número máximo de linhas para qualquer documento do editor. O valor padrão é 0 (0 ou menos é essencialmente infinito) e qualquer outro valor limitará as linhas permitidas. Traço de partida em novas linhas: Ativar ou desativar a auto-indentação para seu código. O padrão está ativado, o que manterá qualquer tabulação definida em uma linha quando uma nova linha for adicionada. Auto travessão após a cinta: Habilite ou desabilite a auto-indentação para seu código ao adicionar as chaves {} . O padrão está em vigor, o qual irá tabular a próxima linha de código quando uma nova pulseira { e uma nova linha é adicionada. Habilitar o preenchimento do código: Habilitar ou desabilitar a janela de conclusão de código. Ao escrever o código e isto estiver ativado, você terá uma janela popup mostrando possíveis funções que você pode usar com base no texto de entrada atual. O padrão está ativado. Colunas de funções autoajustáveis: Habilitar ou desabilitar a adição automática de parênteses () quando você utiliza o autocompletar. Ao escrever o código e isto está habilitado, a IDE adicionará automaticamente os dois colchetes () que uma função requer, colocando o cursor dentro deles se a função leva argumentos, ou colocando o cursor atrás deles se não leva argumentos. Esta característica só funciona quando a função foi selecionada a partir do autocompletar. O default está ativado. Desfazer / Redo Limite de pilha: Aqui você pode definir o limite de desfazer / Redo Limite de pilha. Qualquer coisa abaixo de 0 é essencialmente infinito (limitado pela memória disponível), enquanto valores positivos limitarão a pilha a esse número, removendo qualquer ação em a cauda da pilha, se o limite for atingido. O valor padrão é -1. Destaque para os jogos de seleção: Isto destacará levemente qualquer seção de código que seja duplicada em outro lugar. Assim, se você selecionar uma função, por exemplo, todas as instâncias desta função na janela de código atual serão realçadas também. A inadimplência está em andamento. Codificação/conclusão de código colorido para funções obsoletas: Isto destacará qualquer seção de código que esteja obsoleta. O padrão está em vigor.
