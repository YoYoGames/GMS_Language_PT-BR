Opções de jogoUbuntu Esta seção descreve as diferentes opções disponíveis que controlam como seus projetos de jogo Ubuntu(Linux) serão compilados. As diferentes seções são: Geral Esta aba é onde você define o nome de seu jogo como ele deve ser exibido durante a execução, e também pode fornecer um e-mail de contato e uma descrição (curto e longo), bem como uma URL para a página inicial do jogo ou do desenvolvedor. Aqui você também tem a opção de habilitar Vapor em seu jogo. Se você ativar esta opção, certifique-se de ter o SteamworksSDK instalado (veja aqui), e que você adicionou a identificação do aplicativoSteam nas Opções Gerais do jogo. Finalmente, você tem a opção de Desativar o sistema de arquivo sandbox. Ao verificar isso, você desativará o GameMaker Studio 2 sandboxing para salvar e carregar arquivos, permitindo que seus jogos acessem arquivos de qualquer lugar do sistema que esteja rodando o jogo. Esta opção é desmarcada por padrão, pois recomendamos que você mantenha o sandbox do GameMaker Studio 2 para arquivos em todos os sistemas e, quando marcada, algumas áreas de salvamento/carregamento podem ainda não estar disponíveis, dependendo das permissões de nível de sistema operacional. Para maiores informações na caixa de areia, consulte as páginas do Sistema de Arquivos do GameMaker Studio 2. Gráficos As opções gráficas são aquelas que você deve configurar para determinar como seu jogo irá utilizar a placa gráfica de sua máquina Ubuntu alvo. As seguintes opções estão incluídas para que você possa modificar: Iniciar tela cheia: Se isto for verificado, o jogo começará no modo de tela cheia, caso contrário, começará em tela cheia. Isto é desligado por padrão. Permitir a troca de tela cheia: Com isso, o usuário pode mudar de tela cheia para janela e voltar usando os atalhos padrão do Ubuntu OS. Isto é desligado por padrão. Interpolar as cores entre os pixels: Ativa a interpolação linear, que basicamente "suaviza" os pixels. Para gráficos com pixels nítidos, deve ser desligado, mas Se você tiver uma boa combinação de alfa e gráficos suaves, é melhor deixá-la ligada, por padrão. Cursor de exibição: Quando esta opção estiver ativada, o cursor padrão do SO será mostrado em seu jogo. Se esta estiver desabilitada, então enquanto o mouse estiver sobre a janela do jogo, não haverá cursor visível a menos que você tenha criado um como parte do projeto do jogo (isto é feito por padrão). Use a sincronização para evitar rasgos: Esta sincronização ativa ou desativa a sincronização v-sync. Note que se você tiver um jogo com uma velocidade de sala de 120 e o jogador tiver um monitor com uma taxa de atualização de 60, ligando esta opção, sua velocidade de jogo também será fixada em 60. Esta opção está desligada por padrão. Permitir o redimensionamento da janela: A verificação disto permite ao usuário mudar o tamanho da janela do jogo (a opção Janela sem Fronteiras precisa estar desligada para que isto funcione). Esta opção está desativada por padrão. Escala: Aqui você pode optar por manter a relação de aspecto (assim uma sala de 4:3 será "encaixotada" em um 16:9) ou escalar completamente (esticando a imagem para caber na tela inteira). ADVERTÊNCIA! Adesativação da superfície de aplicação desabilitará todas as opções de escalonamento definidas nas Opções de Jogo do Windows até que ela seja ligada novamente. Veja a superfície de aplicação para mais detalhes. Finalmente, há a opção de definir o tamanho da página de textura. O tamanho padrão (e o mais compatível) é 2048x2048, mas você pode escolher entre 256x256 até 8192x8192. Lá é também um botão marcado Pré-visualização que gerará as páginas de textura para esta plataforma e depois abrirá uma janela para que você possa ver como elas são. Isto pode ser muito útil se você desejar ver como são as páginas de textura estruturada e para evitar ter páginas de textura maiores (ou menores) do que o necessário. Para maiores informações sobre páginas de textura, por favor, veja aqui. NOTA: Esteja ciente de que quanto maior o tamanho da página de textura, menos compatível seu jogo estará nos PCs com especificações mais baixas. Imagens Aqui você pode adicionar uma tela Splash para seu jogo, e definir o ícone do jogo que será usado. O ícone deve ser uma imagem de 64x64 pixels no formato .png, e a tela de projeção deve ter pelo menos o mesmo tamanho da primeira sala de seu jogo (ou sua porta de visualização) e ser no formato .png também. A tela de respingo será mostrado enquanto o jogo carrega, e pode ser ativado ou desativado verificando a opção Use Splash Screen (que está desativada por padrão). Vale notar que o GameMaker Studio 2 possui uma ferramenta Gerador de Imagens de Projeto que pode ser usada para criar automaticamente todas as imagens necessárias para todas as diferentes plataformas alvo de seu jogo está sendo compilado para. Se você usar esta ferramenta, você deve revisar as imagens criadas para garantir que elas sejam o que você precisa.
