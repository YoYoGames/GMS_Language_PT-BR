Opções de jogo As Opções de Jogo são divididas em duas seções, Opções Principais e as Configurações da Plataforma. As opções principais começam com as Opções Gerais, que é onde você pode definir certas coisas que irão afetar o jogo independentemente da plataforma que está sendo projetado para funcionar. A janela tem as seguintes seções: Use DnD™ como Script Type padrão: Quando esta opção for selecionada, todos os novos scripts e eventos de objetos serão abertos usando o editor DnD™em vez do editor de código. Quadros de jogo por segundo: Este valor estabelece o limite do número de quadros por segundo que o GameMaker Studio 2 deve tentar manter seus jogos durante a execução. Este valor representa o número de vezes que o loop do jogo será executado em um segundo (um único loop de jogo é considerado um único "frame") e é definido para um valor padrão de 60, embora 30 também seja freqüentemente utilizado. Há muitos argumentos sobre qual a velocidade da sala é o ideal, mas a resposta real depende de que tipo de jogo que você está fazendo, o tamanho da sala, o número de instâncias e a intensidade do código. Por exemplo, se você tiver uma sala grande e uma velocidade de sala de 30, o jogo pode parecer "agitado" quanto maior for a área que você estiver olhando, é mais fácil para o olho perceber a baixa velocidade da sala, mas se o jogo for intenso, então ajustar a velocidade da sala para 120 pode fazer com que o jogo "atrase" à medida que o fps cai devido à quantidade de processamento sendo feita. Portanto, tome cuidado com este valor e tente sempre ajustá-lo ao que seu jogo precisa. Cor padrão do desenho: A cor padrão para o texto do desenho ou fóruns (e outras coisas) quando nenhuma cor tiver sido definida usando código. Cor padrão do desenho Fora da região da sala: A cor padrão que será usada para limpar o fundo de qualquer área fora da sala de jogos. Gerar imagens do projeto: Clicando aqui, será aberta a ferramenta Gerador de Imagens do Projeto. Data de início do projeto: A data em que o projeto foi iniciado. Tempo de uso do projeto: O tempo decorrido desde que você iniciou o projeto. GUIA DO PROJETO: Este é o identificador único que é criado para cada projeto dentro do GameMaker Studio 2. Se você desejar mudar isso, clique no botão Criar novo GUID para gerar um novo GUID. Gerar mipmaps para páginas de textura separadas: Esta opção fará com que o GameMaker Studio 2 crie mipmaps para aqueles sprites que tenham marcado a opção Página de Textura Separada nas configurações de Textura do Editor Sprite. Habilitando o mipmapping significa que a página de textura original do duende deve ser escalonada e filtrada em múltiplas resoluções dentro do arquivo de textura. Isto é normalmente desligado, mas se você estiver usando vistas para aumentar ou diminuir o zoom em grandes áreas da sala de jogos, ou se estiver trabalhando com um jogo 3D, então você pode querer habilitar isto e depois usar as funções de Mipmapping para ajustar a forma como o jogo lida com os diferentes níveis de mipmap. Observe que se você não estiver sinalizando sprites como utilizando páginas de textura individual, você ainda pode habilitar mipmapping em uma base de grupo por textura do Gerente do Grupo de Textura. Esta opção está desligada por padrão. Steam ID: Se você está criando um jogo que deve ser distribuído através do Steam, então você precisa fornecer o Steam App ID aqui. Certifique-se de ter adicionado o caminho para o SteamworksSDK nas Preferências de Plataforma ao usar esta opção, e garantir que você tenha tudo configurado corretamente no painel de controle da Steamworks. Habilitar o controle da fonte: Se você deseja que o projeto esteja sob SCM (Source Control Management), então você precisa verificar isto. Ele está desligado por padrão (veja aqui para mais informações sobre controle de fonte). Autor: O nome do autor do jogo (ou editora ou estúdio). Projeto Última Alteração: Esta seção simplesmente mostra o carimbo de data/hora pela última vez que o projeto foi alterado (isto não é editável). Aceitar a licença da coluna vertebral: Aqui você pode marcar a caixa para ser mostrada a Licença de Tempo de Espera e aceitá-la. Isto permitirá que você utilize os sprites de espinha em seu jogo (esta licença também será mostrada na primeira vez que você tentar adicionar um sprite de espinha. ao seu projeto e ainda não verificou a licença aqui antes). O Navegador de Ativos também terá outras opções de jogo disponíveis para você, dependendo das plataformas alvo que estão disponíveis para a licença que você possui e nem todas elas podem estar disponíveis ou visíveis.
