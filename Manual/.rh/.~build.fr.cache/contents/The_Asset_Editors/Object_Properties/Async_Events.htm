Os eventos da Async Um Evento Assíncrono é aquele que é disparado quando o GameMaker Studio 2 recebe uma chamada de retorno de alguma fonte externa, que pode ser da web ou do dispositivo que está rodando seu jogo. Basicamente, você diz ao GameMaker Studio 2 para fazer algo - como carregar uma imagem - e ele começará a fazer isso, mas continuará a fazer o que tiver que fazer enquanto estiver trabalhando. Então, quando o pedido for concluído, uma ligação de retorno será enviada para o GameMaker. O Studio 2 e quaisquer Eventos Assíncronos definidos para esse tipo de callback serão demitidos. Observe que os Eventos Assíncronos são disparados para todas as instâncias que os têm, muito parecidos com os eventos do teclado, de modo que você pode - por exemplo - fazer uma chamada http_get() em uma instância, mas ter o Evento HTTP Assíncrono em outra para lidar com a chamada de retorno. NOTA: Você deve estar ciente de que, devido à proteção XSS nos navegadores, as solicitações e tentativas de carregar recursos através de domínios são bloqueadas e podem parecer retornar resultados em branco ao usar qualquer um dos seguintes eventos. Há vários tipos de eventos associados à categoria de eventos assíncronos, e todos eles são explicados nas seções abaixo: NOTA: A variável async_load que é mencionado nas seções seguintes só é válida nestes eventos, pois o mapa DS que ele aponta é criado no início do evento, depois apagado novamente no final, com esta variável sendo redefinida para um valor -1 em todos os outros momentos.
