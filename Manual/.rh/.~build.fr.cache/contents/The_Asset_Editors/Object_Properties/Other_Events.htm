Os outros eventos Esta página explica cada um dos diferentes eventos que são coletados sob a categoria Outros eventos no Editor de Objetos: Sala Exterior Este evento é acionado quando uma instância sai da sala, e é baseado em uma verificação feita contra o duende atribuído (e suas propriedades) da instância, de modo que, mesmo que você tenha definido a escala da imagem x ou y para um valor diferente de um, este evento só será acionado quando o duende inteiro estiver fora da tela. Se a instância não tiver sprite, então a posição da instância é usada e no momento em que sua posição x ou y estiver fora da sala, então ela também acionará o evento. Este evento é normalmente usado para coisas como balas, onde são destruídas uma vez que saem da sala para que você não acabe com milhões de balas voando infinitamente e fazendo com que seu jogo abrande. Note que este evento só é acionado uma vez. quando a instância deixa a sala inicialmente. Intersectar Fronteira Este evento é acionado quando uma instância "toca" a borda interna da sala e, como o evento da sala externa, é baseado em uma verificação feita contra o sprite atribuído da instância, de modo que mesmo que você tenha ajustado a escala da imagem x ou y para um valor diferente de um, este evento ainda será acionado no momento em que uma parte do duende intercepta o limite da sala. Se a instância não tem duende, então a posição da instância é usada e o momento em que sua posição x ou y intercepta o limite da sala, então também vai desencadear o evento. Você pode usar este evento para fazer coisas como dizer a uma instância para "saltar" de volta para dentro em direção à sala, em vez de ir para fora. Vistas As câmeras são normalmente definidas no editor da sala e usadas para mostrar apenas uma pequena área de uma sala grande a qualquer momento. Esta categoria de eventos tem duas sub-categorias separadas(Vista Externa e Intersect Boundary), com oito eventos diferentes em cada um correspondendo aos oito portos de visão disponíveis. Estas duas categorias funcionam exatamente da mesma forma que os eventos das respectivas salas, tomando apenas o limite da vista da câmera como a coisa para a instância a ser verificada em relação em vez da sala. Início do jogo Este evento especial é acionado apenas uma vez em todo o jogo e somente para aquelas instâncias com ações ou códigos colocados no evento. Estas instâncias devem estar presentes na primeira sala do jogo e ter sido colocadas lá no editor da sala, ao invés de do que foram criadas dinamicamente usando código ou ações. Deve-se observar que este evento acontece após o evento de criação de todas as instâncias e, portanto, pode conter código ou ações com variáveis que foram previamente definidas naquele evento. Este evento é tipicamente definido em apenas um objeto "controlador" e normalmente é usado para inicializar variáveis globais, iniciar música, ler de arquivos e qualquer outra coisa que normalmente é feita apenas uma vez no início de um jogo. Fim do jogo Semelhante ao evento Game Start acima mencionado, este evento só é acionado uma vez em todo o jogo e isso pouco antes da janela do jogo fechar de fato. Novamente, por exemplo, com este evento para realmente fazer qualquer coisa que tenha que estar na sala como o jogo está fechado. Por favor, note que este evento só é realmente viável para Windows, Ubuntu (Linux) e macOS jogos como dispositivos móveis pode simplesmente fechar o jogo sem dar ao GameMaker Studio 2 o oportunidade de convocar este evento. Para jogos de navegador feitos com o módulo HTML5, este evento deve funcionar quando a guia do jogo ou navegador estiver fechada, mas nem todos os navegadores suportam este recurso (Firefox, Chrome e Safari suportam no momento de escrever), portanto, o cuidado deve ser ser tomada ao utilizá-la. Normalmente, você usaria este evento para salvar dados. Início do quarto Este evento acontece para todas as instâncias inicialmente em uma sala, quando a sala começa. Observe que isso acontece após o evento de criação e, portanto, pode depender de variáveis e coisas definidas anteriormente nesse evento. Fim do quarto Este evento acontece com todas as instâncias que estão na sala quando a sala termina. Este evento é muito útil para "limpeza" após o término de um nível, por exemplo, pode-se apagar recursos carregados aqui, ou remover um sistema de partículas para evitar a memória. vazamentos, etc... Fim da Animação No GameMaker Studio 2 os sprites não são coisas estáticas, pois podem ter sub-imagens e ser animados em velocidades diferentes. Cada frame de uma animação (chamada de subimagem) tem seu próprio número, começando em 0, que pode ser verificado em código ou através de ações, mas às vezes tudo o que você realmente precisa saber é quando a animação tiver terminado. É quando este evento é acionado, logo no final da animação, quando o índice da subimagem mostra que o último frame foi alcançado. Este evento é realmente útil para muitas coisas, por exemplo, um objeto de explosão onde você pode dar o exemplo para se destruir quando o último frame da animação é atingido. Atualização em Animação Este evento foi projetado somente para uso com as Funções de Animação Esquelética. É um evento especial que é acionado a cada passo em uma instância que usa um duende de animação esquelético, e é projetada para "interceptar" os dados ósseos após a orientação dos ossos ter sido calculada para o estado atual da animação, mas antes que esses dados sejam comprometidos a serem usados para o desenho. Isto permite fazer modificações nos dados do osso usando as funções apropriadas. Evento de Animação Este evento foi projetado somente para uso com as Funções de Animação Esquelética. Este evento é acionado em um objeto que tem um duende de animação esquelético atribuído a ele, e onde o objeto está usando o desenho padrão para o duende atribuído, ou o duende está sendo desenhado via draw_self(). Atualmente, ele não acionará em sprites de animação de esqueleto desenhados por qualquer outro maneira. O evento é projetado para capturar e transmitir dados que foram atribuídos a eventos particulares na animação sprite, conforme definido no editor usado para fazê-los. Quando o Evento de Animação é acionado, será criado um Mapa DS especial chamado de mapa de eventos_dados (muito parecido com o mapa de carga_assíncrona para um evento assíncrono). Este mapa terá todos os seguintes pares de chaves/valores que você pode analisar para obter informações sobre o evento: "name": o nome do evento (conforme definido no programa de animação) "track": o índice da faixa em que a animação está tocando (o padrão é 0) "integer": um valor inteiro associado ao evento (como definido no programa de animação, o padrão é 0) "float": um valor flutuante associado ao evento (como definido no programa de animação, o padrão é 0,0) "string": um valor de string associado ao evento (como definido no programa de animação, o padrão é uma string vazia "") Caminho terminado Os caminhos podem ser uma parte importante de qualquer jogo, seja para um jogo do tipo Tower Defense, ou um shoot-em-up ou qualquer outra coisa, e muitas vezes é realmente importante que uma instância faça alguma ação em particular quando atinge o fim de seu caminho. Este evento detecta isto e é acionado quando acontece permitindo adicionar código ou ações para lidar com as circunstâncias, por exemplo, em um jogo de defesa da torre você usaria isto para destruir a instância e remover uma saúde ponto a partir do objeto objetivo. Eventos de usuários Estes são eventos especiais que não são acionados automaticamente pelo GameMaker Studio 2, mas têm que ser implicitamente chamados por você usando código ou ações enquanto o jogo está em execução. Existem 12 eventos de usuário no total e eles podem conter código ou ações e eles podem ser chamados a qualquer momento no loop do jogo, tornando-os muito úteis para criar seus próprios eventos que acontecem quando você decide. Mensagem de radiodifusão Este evento é acionado sempre que um sprite ou uma seqüência gera uma cadeia de mensagens transmitidas.
