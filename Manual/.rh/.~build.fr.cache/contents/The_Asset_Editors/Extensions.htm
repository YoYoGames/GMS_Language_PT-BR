O Editor de Extensão Um pacote de extensão pode ser usado para adicionar uma série de funções adicionais e constantesà linguagem GameMaker(GML), ou como uma forma de transferência ativos do jogo ou mesmo projetos inteiros facilmente entre as pessoas. Se a extensão é para adicionar funcionalidade extra, então as funções usadas podem vincular seu jogo a SDKsexternos ou código fonte escrito em outras linguagens (como Java para Android, por exemplo), bem como GML nativo. Normalmente, as extensões de terceiros serão do GameMaker Mercado e aqueles que você subscreveu para depois ser listada na janela da Bibliotecado Mercado e pode ser instalada a partir daí. Se você não recebeu seu pacote de extensão do Marketplace, você ainda pode adicioná-lo a um projeto novamente por clicando com o botão direito do mouse em qualquer lugar do Navegador de Ativos e selecionando Adicionar Existente, depois navegando até o local da extensão para selecioná-lo. Se você tiver um *.GMEZ criado com uma versão anterior do GameMaker Studio, você pode adicioná-los selecionando o botão direito do mouse opção do menu Import Extension. OBSERVAÇÃO: O formato legado *.GEX não é mais suportado e não pode ser importado para os projetos do GameMaker Studio 2. Uma vez que você tenha adicionado uma extensão ao Navegador de Ativos, você deve dar-lhe um nome e um número de versão e também definir para quais plataformas você deseja exportá-lo. Ao desmarcar as plataformas selecionadas, você pode dizer ao GameMaker Studio 2 para não exportar a extensão como parte do executável para essas plataformas. Você também deve garantir que as propriedades da extensão correspondam à(s) plataforma(s) alvo, e editar as metas de exportação para os arquivos de extensão com base no seguinte: Todos os alvos: Extensões GML. Estas são extensões feitas usando a linguagem GameMaker(GML) e são compatíveis com todas as plataformas alvo. Windows(Standard, UWP, XBoxOne): Você pode usar arquivos do windows DLL (Dynamic Link Libraries). Observe que se você tiver verificado o "Use x64 Windows Runtime " nas Opções de Jogo do Windows, então todas as DLLs para o projeto devem ser compatíveis com 64bit. As DLLs de 32 bits não funcionarão com o executável de 64 bits. macOS: O alvo macOS pode usar os arquivos padrão DyLib para extensões. Ubuntu (Linux): Para o alvo Ubuntu você precisa ter certeza de que sua extensão utiliza arquivos SO. HTML5: Estas plataformas podem utilizar as extensões JavaScript. Android: Esta plataforma requer uma extensão a ser escrita em Java ou em Android Framework SDK. iOS / tvOS: As extensões para esta plataforma são escritas no Objetivo C e requerem um arquivo *.mm e um arquivo *.h, ou se estiver usando qualquer Frameworks, um arquivo nativo *.a pré-compilado. PlayStation: Para PlayStation 3 ou 4 suas extensões devem ser arquivos PRX, sendo que cada alvo requer um PRX diferente para estar de acordo com a especificação do alvo (seja PS3 ou PS4). Se estiver trabalhando em uma PSVita, a extensão deve ser SUPRX. NOTA: Algumas extensões são necessárias para seguir convenções de nomenclatura específicas para que o GameMaker identifique corretamente a plataforma e a arquitetura da CPU para a qual foram projetadas. Consulte a seção sobre Criação de Extensões para obter mais informações. Como uma única extensão pode ter vários arquivos associados, é possível criar uma única extensão para múltiplas plataformas, neste caso você deseja especificar todas as plataformas compatíveis no editor de extensão principal (por exemplo: marque a caixa de seleção ao lado deles) e depois nas propriedades individuais dos arquivos para cada parte da extensão, definir a plataforma para a qual esses arquivos devem ser exportados. Para mudar a plataforma alvo de um arquivo de extensão específico ou de um espaço reservado, você deve primeiro abrir as propriedades da extensão (clique duas vezes em a extensão no Navegador de Ativos ou use e selecione Open Properties), e então uma vez nas propriedades de extensão selecione o arquivo de recurso da direita que você deseja editar: NOTA: Se você estiver exportando para o alvo macOS ou Ubuntu, então você pode definir a opção "Uncompress as zip file" para preservar as permissões de arquivo para a extensão. Uma vez instalada sua extensão, você pode usar as funções e constantes para ela da mesma forma que você usaria as funções e constantes incorporadas na linguagem GameMaker. Se seu jogo for multiplataforma (por exemplo, ele é projetado para rodar em ambas Windows e Android ), então você deve usar as constantes e funções do sistema operacional para verificar a plataforma antes de chamar funções específicas de extensão para evitar problemas, por exemplo: switch(os_type) { case os_android: call_android_extension_function(x, y, z); break; caso os_windows: call_windows_extension_function(x, y, z); pausa; } Note que para diferentes plataformas suas extensões podem compartilhar nomes de funções, então se você tiver uma extensão com arquivos distintos para (por exemplo) aparecer uma mensagem para Windows e para Android, ambos os arquivos de extensão podem compartilhar o mesmo nome de função por tanto já que ambos estão prontos para exportar apenas para suas respectivas plataformas. Se você quiser criar suas extensões ganhas, então você pode encontrar as informações básicas necessárias nas páginas seguintes: Criando uma extensão Extensões doAndroid ExtensõesiOS / tvOS YoYo Games Extensões de Terceiros As versões antigas do GameMaker tinham muitas funcionalidades de terceiros, mas isso era ineficiente e difícil de manter, já que se alguma funcionalidade específica fosse alterada por terceiros, então toda a IDE GameMaker e Tempo de execução teria que ser atualizado. Assim, as versões atuais adotaram o uso de pacotes de extensões para adicionar suporte de terceiros a seus jogos. Isto significa que você pode adicionar somente aquelas funções extras que você necessita e que você pode até mesmo adaptá-las e editá-las de acordo com suas necessidades específicas. A maioria dessas extensões será baixada e instalada para você quando você selecionar uma opção de jogo específica (como Facebook ou Google Mobile Ads, por exemplo), mas você pode encontrar mais informações sobre elas em a seção apropriada para cada plataforma na Base de Conhecimento dos Jogos YoYo: Plataformas e configuração doGameMaker Studio 2 Você pode encontrar as próprias extensões na seguinte página do Marketplace: YoYo Games Marketplace Ativos Observe que estas extensões vêm com uma demonstração e um arquivo de ajuda (no formato *.pdf ) que explica como utilizá-las, e como tal suas funções não são cobertas pelo manual.
