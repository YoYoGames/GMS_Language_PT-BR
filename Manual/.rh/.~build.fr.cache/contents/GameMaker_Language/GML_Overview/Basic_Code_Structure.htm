Estrutura básica do código O código é escrito em blocos e um bloco de código típico consiste em um conjunto de instruções, chamadas dedeclarações, que são então interpretadas pelo GameMaker Studio 2 e costumava fazer algo acontecer dentro de seu jogo. Esse "algo" pode ser tão simples como adicionar 2 e 2 para obter 4, ou tão complexo como fazer um inimigo fugir quando sua saúde fica abaixo de um certo valor. A estrutura real do programa pode variar muito, dependendo das funções que utiliza, mas, decomposto ao básico, parece apenas assim: ; ; ... As declarações devem ser separadas com um símbolo ';' para evitar erros com declarações variáveis e para manter seu código limpo e arrumado, e podem consistir em declarações variáveis, expressões e chamadas para funções específicas. Você também pode "agrupar" declarações como um bloco usando os colchetes {} para que elas funcionem juntas, como nos seguintes exemplo condicional: se () { ; ; ... } OBSERVAÇÃO: A linguagem GameMaker também aceitará começar em e terminarem em vez dos parênteses {}, embora esta não seja normalmente a forma mais comum de fazê-lo: se () começar ; ; ... fim Aqui está uma representação mais visual de como um bloco de código pode parecer, desta vez criado como um script no Editor de Scripts do GameMaker Studio 2: Há uma série de diferentes tipos de declarações, expressões, condicionamentos e funções, todas elas discutidas em seções subseqüentes do manual.
