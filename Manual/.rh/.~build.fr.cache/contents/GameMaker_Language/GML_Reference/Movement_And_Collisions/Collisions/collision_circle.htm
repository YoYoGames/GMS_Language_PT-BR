círculo_de_colisão Collision_circle cria uma zona circular do raio dado pelo usuário em torno dos pontos x1,y1 e, em seguida, verifica essa zona para uma colisão com qualquer instância do objeto especificado pelo argumento "obj". Esta verificação pode ser precisa ou não, mas para que colisões precisas sejam habilitadas, o objeto ou instância que você está verificando também deve ter colisões precisas habilitadas para seu sprite. Caso contrário, a verificação padrão é baseada em caixas delimitadoras. A imagem a seguir ilustra como isto funciona: Aqui, a instância no meio está usando um círculo de colisão para verificar a bola objetos. Agora, os azuis não têm uma caixa de delimitação precisa e, como você pode ver, mesmo que o duende não esteja realmente tocando o círculo, a colisão ainda pode acontecer (mesmo se você definir a opção precisa na função como verdadeira) como a delimitação A caixa daquele duende sobrepõe a área circular definida pelo círculo de colisão. Por outro lado, as bolas verdes só serão consideradas em colisão se o duende real sobrelaçar com o círculo definido. Lembre-se, para colisões precisas a serem consideradas tanto o objeto sprite quanto a função de colisão devem ter uma marcação precisa como em cima. Também deve ser observado que o valor de retorno da função pode ser o id de qualquer uma das instâncias consideradas como em colisão. Note que ao invés de um índice de objetos você pode fornecer uma instância id para verificar uma instância específica, ou as palavras-chave da instância all, ou other (dependendo do evento e do escopo do código atual). Sintaxe: collision_circle(x1, y1, rad, obj, prec, notme); Argumento Descrição x1 A coordenada x do centro do círculo a verificar. y1 A coordenada y do centro do círculo a verificar. rad O raio (distância em pixels de seu centro até sua borda). obj O objeto a ser verificado, por exemplo, colisões. prec Se a verificação é baseada em colisões pixel-perfect (verdadeiro = lento) ou em sua caixa de delimitação em geral (falso = rápido). notme Se a instância de chamada, se relevante, deve ser excluída(verdadeira) ou não(falsa). Devoluções: Identificação da instância ou de ninguém Exemplo: if collision_circle(x, y, 20, obj_Cursor, false, true) { image_index = 1; } outra imagem_index = 0; O código acima verificará uma circular com um raio de 20pixel para uma colisão com "obj_Cursor" e, se houver uma, definirá o índice da imagem do objeto executando o código para 1, mas se não houver, definirá o índice da imagem do objeto a 0.
