ponto_de_colisão Collision_point verifica o ponto especificado pelos argumentos x1,y1 para uma colisão com qualquer instância do objeto especificado pelo argumento "obj". esta verificação pode ser precisa ou não, mas para para que as colisões sejam habilitadas, o objeto ou instância que você está verificando também deve ter as colisões precisas habilitadas para seus duendes. Caso contrário, a verificação padrão é baseada em caixas delimitadoras. A imagem a seguir ilustra como isto funciona: Lembre-se, para que colisões precisas sejam consideradas tanto o objeto sprite e a função de colisão deve ter uma marcação precisa como em. Também deve ser observado que o valor de retorno da função pode ser o id de qualquer uma das instâncias consideradas em colisão, portanto, se três instâncias se sobrepuserem nesse ponto, qualquer uma de suas identificações poderia ser o valor de retorno da função. Note que ao invés de um índice de objetos você pode fornecer uma instância id para verificar uma instância específica, ou as palavras-chave da instância all, ou other (dependendo do evento e do escopo do código atual). Sintaxe: collision_point(x, y, obj, prec, notme); Argumento Descrição x A coordenada x do ponto a ser verificado. y A coordenada y do ponto a verificar. obj O objeto a ser verificado, por exemplo, colisões. prec Se a verificação é baseada em colisões precisas(verdadeiro, que é mais lento) ou em sua caixa de delimitação em geral(falso, mais rápido). notme Se a instância de chamada, se relevante, deve ser excluída(verdadeira) ou não(falsa). Devoluções: Identificação da instância ou de ninguém Exemplo: if collision_point(x, y, obj_Cursor, false, true) { Pontuação += 10S; } Aqui estamos verificando o ponto na posição do objeto que tem o código para o objeto "obj_Cursor".
