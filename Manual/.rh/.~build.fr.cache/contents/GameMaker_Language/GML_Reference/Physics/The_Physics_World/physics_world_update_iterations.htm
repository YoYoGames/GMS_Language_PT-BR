física_mundo_actualização_actualização_data Tudo no GameMaker Studio 2 é baseado em etapas e, normalmente, cada evento só acontece uma vez por etapa. No entanto, para obter a precisão necessária com as funções físicas, são feitos vários cálculos em cada etapa que são chamados "iterações", cujo número é controlado por esta função. O número padrão para o sistema de física é 10, mas este pode ser alterado para maior ou menor dependendo de quantas vezes você quiser que a física calcule cada passo, mas é recomenda-se que não seja inferior a 5 e não superior a 30. Você também deve estar ciente de que esta função depende do physics_world_update_speed() função. Sintaxe: physics_world_update_iterations(iterações) Argumento Descrição y1 as iterações (por etapa) que o sistema de física realizará Devoluções: N/A Exemplo: physics_world_update_iterations(20); O código acima define o mundo da física na sala atual para realizar cálculos 20 vezes por passo.
