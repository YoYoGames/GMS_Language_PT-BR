get_string_async Esta função abre uma janela e exibe uma mensagem assim como um espaço para o usuário inserir uma string (que conterá a string padrão fornecida para começar). Esta é uma função assíncrona, e como tal o GameMaker Studio 2 não bloqueia o dispositivo em que está sendo executado enquanto espera por uma resposta, mas continua a executar eventos como de costume. Uma vez que o usuário tenha digitado sua seqüência e pressionado o botão "Ok", um evento de diálogo assíncrono é acionado que, somente durante a duração desse evento, terá um mapa DS armazenado na variável async_load. Este mapa conterá as três chaves, " id"," status" e "resultado". "id" é ovalor que foi retornado pela função quando chamado, o "status" irá ser verdadeiro para o botão "Ok" sendo pressionado, ou falso se a mensagem foi cancelada (onde aplicável, pois nem todas as plataformas alvo permitem que a mensagem seja cancelada). Finalmente, o "resultado" retornará a seqüência que a entrada do usuário (ou um fio vazio "" se nenhum foi fornecido). NOTA: Esta função é apenas para depuração e testee não deve ser usada em jogos lançados. Para isso, você deve criar uma interface de usuário personalizada para receber informações dos jogadores, para que você tenha controle total sobre a aparência e o comportamento dos diálogos. Sintaxe: get_string_async(string, default); Argumento Descrição Cordão A mensagem a ser mostrada no diálogo. padrão A cadeia padrão.
