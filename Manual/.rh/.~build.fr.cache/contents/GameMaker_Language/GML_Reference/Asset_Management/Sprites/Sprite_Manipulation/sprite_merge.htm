sprite_merge Esta função fundirá o sprite indexado no argumento 1 ("ind2") com o indexado no argumento 0 ("ind1"). As imagens em si NÃO são fundidas, mas sim os índices de imagem são fundidas, com as sub imagens da sprite "ind2" anexadas às da sprite "ind1", ou seja: elas são adicionadas no final. Observe que se os sprites são de tamanhos diferentes, então os sprites anexados são esticados para caber no tamanho da imagem para "ind1". Esta mudança é permanente, e a partir do momento em que você usar esta função até que o jogo seja fechado ou o duende seja excluído, o duende que está sendo fundido será alterado, porém o duende com o qual está sendo fundido permanecerá o mesmo. NOTA: Nem o bem de imagem que está sendo fundido, nem o bem de imagem que é para fundir, pode ser um dos recursos originais do jogo. Você deve usar sprite_duplicate() primeiro para criar dois novos recursos (se você estiver fundindo ativos de jogos incluídos) e depois fundir esses recursos. NOTA: Esta função é útil apenas para sprites bitmap e não funcionará com sprites SWF ou JSON (Espinha). Sintaxe: sprite_merge(ind1, ind2); Argumento Descrição ind1 O índice do duende a ser fundido. ind2 O índice do duende com o qual a ind1 deve ser fundida. Devoluções N/A Exemplo: var tspr; tspr[0] = spr_add(working_add(working_directory + "Sprite Assets\explode1.png", 16, true, true, 0, 0); tspr[1] = sprite_duplicate(spr_Explosion); sprite_merge(tspr[0], tspr[1]); sprite_index = t_spr[0]; sprite_delete(tspr[1]); O código acima carrega um duende em uma variável local e depois o funde com outro duende que tenha sido duplicado. Finalmente, o sprite fundido é atribuído à instância sprite_index enquanto o sprite carregado é removido. para liberar a memória que utiliza. Observe que em algum momento após esta operação, como nos eventos Destroy ou Room End, você também precisará usar sprite_delete() para remover o duende fundido de memória também.
