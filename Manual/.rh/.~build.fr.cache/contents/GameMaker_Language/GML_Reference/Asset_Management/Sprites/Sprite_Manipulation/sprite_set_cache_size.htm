sprite_set_cache_size_s Ao realizar a mistura de imagens, o HTML5 não pode fazê-lo dinamicamente da mesma forma que um executável poderia ser executado. Portanto, o GameMaker Studio 2 tem que salvar temporariamente uma cópia misturada da imagem e carregá-la. Esta função define quantas cópias mescladas do duende dado pode ser colocado em cache antes que os antigos sejam sobregravados. O padrão é 4. Isto é aplicado a todas as subimagens do duende. Sintaxe: sprite_set_cache_size_size(ind, max); Argumento Descrição ind O índice do sprite para alterar o tamanho do cache de. max O número máximo de cópias em cache do sprite que podem ser armazenadas. Devoluções N/A Exemplo: sprite_set_cache_size(sprite0, 2); Isto irá definir o cache sprite de todas as subimagens de sprite0 para 2 cópias.
