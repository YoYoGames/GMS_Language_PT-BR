window_get_visible_rects Com esta função você pode encontrar a região sobreposta do retângulo definido por (x1,y1) a (x2,y2) em cada um dos displays anexos. A função retornará um array com 8 valores por display (ou seja: se você tiver dois displays, o array terá um comprimento de 16 índices), onde os valores [0 ... 3] correspondem à sobreposiçãox1, sobreposição1, sobreposiçãox2, sobreposição2 - definindo a região de sobreposição nesta tela e serão definidos para 0,0,0,0,0 se não houver sobreposição - e os valores [4 ... 7] correspondem à monitorx1, monitory1, monitorx2, monitory2 - as coordenadas da exibição no espaço de exibição virtual. Isto pode ser usado para testar se uma posição salva da janela será visível ou não (o usuário pode ter desconectado um monitor externo ou moveu a janela para fora da tela que deixou a posição da janela que foi salva como não sendo válida), por exemplo. Sintaxe: window_get_visible_rects(x1, y1, x2, y2); Argumento Descrição x1 A borda esquerda do retângulo para verificar y1 A borda superior do retângulo a ser verificado. x2 A borda direita do retângulo para verificar y2 A borda inferior do retângulo a ser verificado. Devoluções: Array Exemplo: var wx = window_get_x(); var wy = window_get_y(); var ww = window_get_width(); var wh = window_get_height(); display_data = window_get_visible_rects(wx, wy, wx + ww, wy + wh); display_num = array_length(display_data) / 8; O código acima irá gerar uma matriz 1D mantida na variável display_data contendo as informações sobre as telas, bem como criar a variável display_num para manter o número de telas ativas encontradas.
