ds_list_destroy Esta função removerá da memória a estrutura de dados da lista dada, liberando os recursos que estava usando e removendo todos os valores que continha. Esta função deve ser sempre utilizada quando você terminar de usar o ds_list para evitar vazamentos de memória. que pode diminuir a velocidade e quebrar seu jogo. Note que a destruição de uma lista desreferenciará quaisquer estruturas de dados nela armazenadas dando um vazamento de memória, então você precisaria passar pela lista e destruir todos os itens da estrutura de dados manualmente antes de destruí-la. para evitar isso. A única vez que isto não é necessário é quando você tiver assinalado qualquer item da lista como outra lista DS ou como um mapa DS, caso em que estes itens serão destruídos e seus memória limpa automaticamente também. IMPORTANTE! Quando você cria uma estrutura de dados, o valor do índice para identificá-lo é um valor inteiro começando em 0. Isso significa que estruturas de dados de diferentes tipos podem ter o mesmo valor de índice, portanto, em caso de dúvida, você deve estar usando o ds_exists() antes de acessá-los. Observe também que os índices são reutilizados, de modo que um valor de índice de estrutura de dados destruída pode ser usado por um índice recém-criado posteriormente, por isso recomendamos sempre definindo a variável que manteve o índice DS em -1 após a destruição. Sintaxe: ds_list_destroy(id); Argumento Descrição id A identificação da estrutura de dados a ser removida. Devoluções: N/A Exemplo: se (vidas == 0) { ds_list_destroy(AI_list); AI_list = -1; room_goto(rm_Menu); } O código acima verificará o valor da variável global "vidas" construída e, se for 0, destruirá a lista DS indexada na variável "AI_list" e, em seguida, mudará de sala.
